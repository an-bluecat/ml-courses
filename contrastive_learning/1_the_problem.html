Contrastive Learning — Chapter 1: The Problem

What contrastive learning solves — and why supervised learning can't do it alone

Supervised Learning Needs Labels

The label cost problem

TaskWhat a label looks likeCost per 1M examples
Image classification"This is a cat"~$50,000
Medical imaging"This scan shows a tumor"~$5M (needs radiologist)
Sentence similarity"These two mean the same thing"~$200,000

The internet has billions of unlabeled images. Training on 1% of them with labels costs millions.

What supervised learning produces

// ImageNet classifier: 1000 fixed classes image → encoder → [0.0, 0.0, 0.95, 0.0, ...] // "95% cat"   // Problem 1: only knows the 1000 classes it was trained on // Problem 2: the output encodes "is it a cat?" not "what IS it?" // Problem 3: no way to ask "how similar are these two images?"

What we actually want — an embedding space

A function that maps any input to a point in space such that similar things are close, dissimilar things are far. No labels needed to build this — just pairs.

dogs
cars
similar = close  ·  different = far  ·  no class labels needed

Pull Together, Push Apart

One principle, two operations

You don't need class labels. You just need to know which pairs are "the same thing" and which are "different things."

Where do positive pairs come from without labels?

Data augmentation. Take one image, create two different views of it. Same underlying content, different surface appearance.

OriginalView 1 (augmented)View 2 (augmented)
Cat photoCat photo, rotated 15°Cat photo, color-jittered
Cat photoCat photo, cropped top-leftCat photo, cropped bottom-right

The network doesn't know they came from the same image. If it learns to map both views to the same point in embedding space, it must have learned something meaningful about what a "cat" is — with zero labels.

The setup in code

// For every image x in your dataset: x_i = augment(x) // view 1 x_j = augment(x) // view 2 — x_i and x_j are the positive pair   // Every OTHER image in the batch = negative // Batch of 256 images → 256 positives + 254 negatives per anchor   // You don't know class labels. // You only know: x_i and x_j came from the same original image.

The Loss Function — InfoNCE

Step 1 — cosine similarity

Two embeddings. How similar? Use cosine similarity — dot product after normalizing to unit length.

// Two 2D unit-vector embeddings: e_cat1 = [0.6, 0.8] // cat view 1 e_cat2 = [0.7, 0.714] // cat view 2 (positive pair) e_dog = [−0.8, 0.6] // dog (negative) e_car = [0.0, −1.0] // car (negative)   // Cosine similarity = dot product (unit vectors): sim(cat1, cat2) = 0.6×0.7 + 0.8×0.714 = 0.42 + 0.57 = +0.99 // very similar sim(cat1, dog) = 0.6×−0.8 + 0.8×0.6 = −0.48+0.48 = 0.00 // orthogonal sim(cat1, car) = 0.6×0.0 + 0.8×−1.0 = 0 + (−0.8) = −0.80 // opposite
How does unit normalization work? (expand)
// Raw vector v = [3.0, 4.0]   // Step 1 — length (L2 norm): ||v|| = sqrt(3.0² + 4.0²) = sqrt(9 + 16) = sqrt(25) = 5.0   // Step 2 — divide each component: v_unit = [3.0/5.0, 4.0/5.0] = [0.6, 0.8]   // Step 3 — verify unit length: ||v_unit|| = sqrt(0.6² + 0.8²) = sqrt(0.36 + 0.64) = sqrt(1.0) = 1.0 ✓   // Why this matters: cosine similarity = (a·b) / (||a||×||b||) // If both unit vectors: denominator = 1×1 = 1, so dot product = cosine sim directly   // Applied to cat2 = [7.0, 7.14]: ||cat2_raw|| = sqrt(49 + 50.98) = sqrt(99.98) ≈ 10.0 cat2_unit = [7.0/10, 7.14/10] = [0.7, 0.714]   dot(cat1_unit, cat2_unit) = 0.6×0.7 + 0.8×0.714 = 0.42 + 0.571 = 0.99

Step 2 — scale by temperature τ, apply softmax

// Temperature τ = 0.1 (small = sharper distribution) τ = 0.1   // Scaled similarities: sim/τ: cat2=0.99/0.1=9.9, dog=0.0/0.1=0.0, car=−0.8/0.1=−8.0   // Softmax = probability of picking each as the correct positive: exp(9.9) = 19930, exp(0.0) = 1.0, exp(−8.0) = 0.0003 sum = 19931   P(cat2 | cat1) = 19930/19931 = 0.9999 // almost certain P(dog | cat1) = 1/19931 = 0.00005

Step 3 — InfoNCE loss (cross-entropy on this distribution)

// How surprised are we that cat1's match is cat2? loss = −log P(correct positive | anchor) = −log( exp(sim(i,j)/τ) / Σ_k exp(sim(i,k)/τ) )   // Good embedding (positive very similar to anchor): loss = −log(0.9999) = 0.0001 // tiny   // Bad embedding (positive looks same as negatives): // sim(i,j) ≈ sim(i,k) for all k → P = 1/N loss = log(4) = 1.39 // "random guessing"

InfoNCE is literally cross-entropy from the info theory course — the model is penalized for being surprised that its positive match is the positive. Same math, same intuition.
