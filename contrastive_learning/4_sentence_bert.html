Contrastive Learning — Chapter 4: Sentence-BERT

Contrastive learning for sentences

Why Vanilla BERT Is Bad at Sentence Similarity

BERT requires both sentences at the same time

// To compare sentences A and B, BERT needs: input = "[CLS] sentence_A [SEP] sentence_B [SEP]" // Cross-attention between A and B similarity_score = BERT(concat(A, B))   // Problem: to find the most similar sentence in a 10,000-sentence corpus: // Must run BERT on all 10,000 pairs = 10,000 forward passes // Each pass: ~500ms on CPU = 83 minutes for one query // Completely unusable for search/retrieval at scale.

Sentence-BERT's fix — precompute embeddings

// Sentence-BERT trains BERT to produce a single vector per sentence: emb_A = SBERT(sentence_A) // → 768-dim vector ← precomputed once emb_B = SBERT(sentence_B) // → 768-dim vector ← precomputed once   // Similarity = cosine sim (just a dot product): sim(A, B) = dot(emb_A, emb_B) // microseconds   // Now: find most similar in 10,000 sentences: // Precompute 10,000 embeddings (once) // Query: 1 embedding + 10,000 dot products = ~5ms // Same accuracy, 65ms vs 83 minutes. 3000x speedup.

Where Positive Pairs Come From — NLI Data

Natural Language Inference (NLI) dataset

NLI datasets label pairs of sentences as: entailment (same meaning), contradiction (opposite), or neutral. Sentence-BERT uses these as free contrastive supervision.

PremiseHypothesisLabelUsed as
"A dog is running in the park""An animal is in the park"entailmentpositive pair ↔ pull together
"A dog is running in the park""A cat is sleeping on the couch"contradictionhard negative ↔ push apart (harder)
"A dog is running in the park""The weather is sunny today"neutralsoft negative ↔ push apart (softer)

550,000 sentence pairs from SNLI + MultiNLI datasets — free, pre-labeled, no augmentation needed.

Computing the loss — concrete example

// Anchor: "A dog is running in the park" emb_anchor = SBERT("A dog is running in the park") emb_positive = SBERT("An animal is in the park") // entailment emb_hard_neg = SBERT("A cat is sleeping on the couch") // contradiction   // After training, similarities: sim(anchor, positive) = +0.85 // should be high sim(anchor, hard_neg) = −0.30 // should be low   // Loss (InfoNCE with τ=0.05): numerator = exp(0.85/0.05) = exp(17) = 24,000,000 denominator = exp(17) + exp(−6) = 24,000,000 + 0.0025 ≈ 24,000,000 loss = −log(24M / 24M) ≈ 0.0 // almost zero — great separation

What Sentence-BERT Powers

Semantic search — find relevant docs

// Query: "how to fine-tune a language model" query_emb = SBERT("how to fine-tune a language model")   // Compare against your corpus (precomputed): sim(query, "transfer learning tutorial") = 0.82 sim(query, "introduction to neural networks") = 0.55 sim(query, "cooking pasta recipe") = 0.03   // Return top-k by cosine similarity. No keyword matching needed.

RAG (Retrieval-Augmented Generation)

// The pipeline powering most LLM applications:   User: "What was OpenAI's revenue in 2024?"   // Step 1: SBERT encodes the question q_emb = SBERT("OpenAI revenue 2024")   // Step 2: Find top-3 similar chunks from your knowledge base chunks = retrieve_top_k(q_emb, k=3) // Returns: ["OpenAI reported $3.4B revenue in 2024...", ...]   // Step 3: LLM generates answer grounded in retrieved chunks answer = LLM(question + chunks) // "According to the reports, OpenAI's 2024 revenue was $3.4B..."

Clustering, deduplication, recommendation

// Cluster 10,000 customer support tickets by topic: embeddings = [SBERT(ticket) for ticket in tickets] clusters = KMeans(embeddings, k=20) // Each cluster = a support topic. No manual labeling.   // Find duplicate articles in a news database: // If sim(article_a, article_b) > 0.95 → likely duplicate   // "Users who read X also liked Y" recommendation: // Recommend articles with similar embeddings to what user read
