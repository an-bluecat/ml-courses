Diffusion Ch1: The Idea

Destroy an image with noise, then learn to reverse it

The Big Idea

The generative modeling problem

You have a dataset of images — say, 10 million photos of cats. You want a model that can generate new cat images that weren't in the training set. The images should look real, not blurry or distorted.

Other approaches: GANs (hard to train — mode collapse, unstable), VAEs (blurry outputs — too much compression). Diffusion models produce sharper images, are more stable to train, and have a cleaner mathematical foundation.

The diffusion insight — two processes

Forward process (easy)

Add a tiny bit of Gaussian noise to an image. Repeat 1000 times. The image gradually becomes pure random noise. You can compute this without a neural network.

Reverse process (learned)

Train a neural network to remove a tiny bit of noise from a noisy image. If you can do that, you can generate by starting from pure noise and removing noise 1000 times.

At t=1000 the image is indistinguishable from N(0,I). At generation time, we start from here and run the arrow backwards.

Why not just train on x→noise directly?

If you tried to train a network to go straight from noise to image in one step, it would need to memorize all possible images — impossible. The key is breaking it into 1000 tiny steps, each of which is a small, learnable denoising. Each step removes only a little noise, so the task is well-defined.

Forward Process — Adding Noise Mathematically

The noise schedule

At each step t, we define a noise level β_t (small, between 0 and 1). We set α_t = 1 − β_t (how much of the signal survives). Typical values: β ranges from 0.0001 at t=1 to 0.02 at t=1000 (linearly).

// Forward step: add noise at step t x_t = sqrt(α_t) × x_{t-1} + sqrt(1 − α_t) × ε where ε ~ N(0, I) // fresh noise each step   // α_t = 1 − β_t (fraction of signal kept) // β_t ≈ 0.001 (small noise per step)

Concrete: trace one pixel through 3 steps

// Starting pixel value: x_0 = 0.8 (bright pixel, range -1 to 1) // β_1 = 0.001 → α_1 = 0.999 // β_2 = 0.001 → α_2 = 0.999 // β_3 = 0.001 → α_3 = 0.999   // Step 1: (assume ε₁ = +1.0 from N(0,1)) x_1 = sqrt(0.999) × 0.8 + sqrt(0.001) × 1.0 = 0.9995 × 0.8 + 0.03162 × 1.0 = 0.7996 + 0.0316 = 0.8312   // Step 2: (assume ε₂ = -0.5) x_2 = sqrt(0.999) × 0.8312 + sqrt(0.001) × (−0.5) = 0.9995 × 0.8312 + 0.03162 × (−0.5) = 0.8308 − 0.0158 = 0.8150   // Step 3: (assume ε₃ = +0.3) x_3 = sqrt(0.999) × 0.8150 + sqrt(0.001) × 0.3 = 0.8146 + 0.0095 = 0.8241   // After 3 steps: pixel moved from 0.8 to 0.824 // Still mostly the original value — noise is tiny per step

After many steps — signal decays exponentially

// What fraction of x_0 survives after t steps? // Each step multiplies by sqrt(α). After t steps: sqrt(α_1) × sqrt(α_2) × ... × sqrt(α_t) signal_fraction = sqrt(ᾱ_t) where ᾱ_t = α_1 × α_2 × ... × α_t   // With β_t ≈ 0.001 for all steps: ᾱ_100 = 0.999^100 = 0.905 // 90% signal, 10% noise ᾱ_500 = 0.999^500 = 0.607 // 61% signal, 39% noise ᾱ_1000 = 0.999^1000 = 0.368 // 37% signal   // Real DDPM uses a variance schedule that goes further: ᾱ_1000 ≈ 0.0001 // basically pure noise by t=1000

The Closed Form — Jump Directly to Any Noise Level

We don't need to apply 500 steps to get x_500

Because each forward step is Gaussian, the composition of many Gaussian steps is also Gaussian. This gives a closed-form expression that jumps directly from x_0 to x_t in one shot.

// Key reparameterization (DDPM paper, Ho et al. 2020): x_t = sqrt(ᾱ_t) × x_0 + sqrt(1 − ᾱ_t) × ε where ε ~ N(0, I)   // ᾱ_t = product of all α_1 × α_2 × ... × α_t // No loop needed — compute x_t directly from x_0

Concrete: x_0=0.8, compute x_500 directly

// DDPM linear schedule: ᾱ_500 ≈ 0.10 ᾱ_500 = 0.10 // 10% signal remains at step 500   // Sample a noise vector: ε ~ N(0,1) // Say ε = 0.43 (one random draw)   x_500 = sqrt(0.10) × 0.8 + sqrt(1 − 0.10) × 0.43 = sqrt(0.10) × 0.8 + sqrt(0.90) × 0.43 = 0.3162 × 0.8 + 0.9487 × 0.43 = 0.253 + 0.408 = 0.661   // Interpretation: x_500 is mostly noise (0.408 from noise) // with a faint signal contribution (0.253 from original 0.8)

What this means for training

The closed form means we can train efficiently. For each training sample x_0:

  1. Sample a random timestep t ~ Uniform(1, 1000)
  2. Sample a noise vector ε ~ N(0, I)
  3. Compute x_t = sqrt(ᾱ_t) × x_0 + sqrt(1−ᾱ_t) × ε (closed form, one line)
  4. Pass x_t and t to the network; ask it to predict ε
  5. Loss = ||ε − ε_θ(x_t, t)||²

No loop over 1000 steps per training example. Each example uses one random t. Fast and parallelizable.

x_1000 directly — pure noise with no memory of x_0

// ᾱ_1000 ≈ 0.0001 (DDPM schedule) x_1000 = sqrt(0.0001) × x_0 + sqrt(0.9999) × ε = 0.01 × 0.8 + 0.9999 × ε = 0.008 + 0.9999 × ε   // The x_0 contribution is 0.008 — essentially zero // x_1000 ≈ N(0, 1) — pure Gaussian noise, origin forgotten Var(x_1000) ≈ 0.0001 × Var(x_0) + 0.9999 ≈ 1.0

Reverse Process — The Neural Network

What the network learns

Train a U-Net ε_θ(x_t, t) that takes a noisy image x_t and the timestep t, and predicts the noise ε that was added. The network must learn: "given this noisy image at noise level t, what noise vector was mixed in?"

// Training objective (simplified DDPM loss): L = E[||ε − ε_θ(x_t, t)||²]   // ε: the actual noise we added (we know this — it's the label) // ε_θ(x_t, t): the network's predicted noise // Minimize MSE between real noise and predicted noise

Why predicting noise works — intuition

// We know: x_t = sqrt(ᾱ_t) × x_0 + sqrt(1−ᾱ_t) × ε Rearranged for x_0: x_0 = (x_t − sqrt(1−ᾱ_t) × ε) / sqrt(ᾱ_t)   // If the network can predict ε, we can recover x_0! // Concrete: x_t=0.661, ᾱ_t=0.10, network predicts ε̂=0.43 x̂_0 = (0.661 − sqrt(0.90) × 0.43) / sqrt(0.10) = (0.661 − 0.9487 × 0.43) / 0.3162 = (0.661 − 0.408) / 0.3162 = 0.253 / 0.3162 = 0.800 // recovered the original!

Sampling — generating new images from scratch

// Start from pure noise x_T ~ N(0, I) // e.g. x_T = [0.43, -1.21, 0.87, ...]   // Iterative reverse: apply the denoising formula T times for t in range(T, 0, -1): ε̂ = ε_θ(x_t, t) // network predicts noise x̂_0 = (x_t − sqrt(1−ᾱ_t)×ε̂) / sqrt(ᾱ_t) // estimate clean image x_{t-1} = sqrt(ᾱ_{t-1}) × x̂_0 + sqrt(1−ᾱ_{t-1}) × ε̂ // re-noise at level t-1 // (+ small stochastic term for DDPM)   // After 1000 iterations: x_0 is a new, never-before-seen image

Why does the network know what to generate?

At training time, the network saw millions of (x_t, t, ε) triples across all noise levels. It learned: at low noise (t≈0), small adjustments; at high noise (t≈1000), predict rough structure. Starting from noise, each reverse step moves slightly toward the learned data distribution.

The network never explicitly told "generate a cat." It just learned "at this noise level, the most likely clean image underlying this pattern looks like...". Emergently, it learns to generate images from the training distribution.
