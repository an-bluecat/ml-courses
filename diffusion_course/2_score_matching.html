Diffusion Ch2: Score Matching

The math behind why noise prediction works

Score Functions — Gradients of Probability

What is a score function?

// The score is the gradient of the log probability with respect to x s(x) = ∇_x log p(x)   // p(x) is the probability density of the data (e.g., images) // log p(x) is higher for more probable inputs (real images) // ∇_x is the gradient with respect to the image pixels   // Interpretation: the score points in the direction that // makes x look more like a real image.

Concrete: 1D Gaussian score

// Suppose p(x) = N(μ=2, σ²=1) — data centered at 2 log p(x) = −0.5 × (x − 2)² + const score s(x) = d/dx log p(x) = −(x − 2) / 1 = 2 − x   // Evaluate at different x values: x = 0: s(0) = 2 − 0 = +2.0 // points right toward μ=2 x = 1: s(1) = 2 − 1 = +1.0 // points right, weaker x = 2: s(2) = 2 − 2 = 0.0 // at the mode — no gradient x = 3: s(3) = 2 − 3 = −1.0 // points left back to μ=2 x = 5: s(5) = 2 − 5 = −3.0 // far from mode: strong pull

The score always points toward the region of highest probability. If you follow the score function, you walk toward the most likely values.

Langevin dynamics — generating samples by following the score

// If you know s(x) = ∇_x log p(x), you can generate samples: x_{t+1} = x_t + η × s(x_t) + sqrt(2η) × z where z ~ N(0, I)   // η = step size (small). First term: follow the score (toward high p). // Second term: add noise to explore and avoid getting stuck.   // Concrete: start at x=5, η=0.1, s(x)=2−x (Gaussian score) step 1: x = 5.0 + 0.1×(−3.0) + noise = 5.0 − 0.30 + noise = 4.7 + noise step 2: x ≈ 4.7 + 0.1×(−2.7) + noise = 4.43 + noise step 10: x ≈ 2.6 // getting close to μ=2 step 100: x ≈ 2.05 // converged, with fluctuations around μ=2

The catch: we can't compute the score of real image data

p(x) for natural images is an incredibly complex, unknown distribution over millions of dimensions. We have samples (the training images), but we can't write down p(x) explicitly, so we can't compute ∇_x log p(x) directly. Score matching is a way to estimate the score from data.

Denoising Score Matching — Estimating Scores from Data

The score of a Gaussian is simple

// For noisy data: x_noisy = x_0 + σ × ε, ε ~ N(0,I) // The conditional distribution: p(x_noisy | x_0) = N(x_0, σ²I)   // Score of this conditional: ∇_x log p(x_noisy|x_0) = (x_0 − x_noisy) / σ² = −ε / σ // negative noise divided by std   // Concrete: x_0=0.8, σ=0.5, ε=0.4, x_noisy=0.8+0.5×0.4=1.0 score = (0.8 − 1.0) / 0.5² = −0.2 / 0.25 = −0.8 // Points left (−0.8) back toward the clean x_0=0.8 ✓

Denoising score matching — the key theorem

Vincent (2011) showed: to estimate the score of the data distribution, it's enough to train a network to denoise noisy samples. Minimizing the denoising error is equivalent to minimizing the score matching objective.

// Score matching objective (what we want, but can't compute): L_SM = E[||s_θ(x) − ∇_x log p(x)||²]   // Denoising score matching (equivalent, computable!): L_DSM = E[||s_θ(x_noisy, σ) − (x_0 − x_noisy)/σ²||²]   // They differ by a constant — minimizing one minimizes the other // We can compute L_DSM because we made the noise ourselves

Connecting score to noise

// From DDPM: x_t = sqrt(ᾱ_t)×x_0 + sqrt(1−ᾱ_t)×ε // Rearrange: x_0 − x_t = -(sqrt(1−ᾱ_t)/sqrt(ᾱ_t)) × ε + (1 - sqrt(ᾱ_t))×x_0   // The key relationship: score ∝ −ε / sqrt(1−ᾱ_t) // score is proportional to negative noise   // Predicting score ↔ predicting noise (up to a known scaling factor) // s_θ(x_t, t) = −ε_θ(x_t, t) / sqrt(1−ᾱ_t)   // Concrete: t=500, ᾱ_500=0.1, predicted ε̂=0.43 score estimate = −0.43 / sqrt(1−0.1) = −0.43 / 0.949 = −0.453 // This points toward cleaner images (away from the noise direction)

Why not train the score directly?

The score function s(x) is a vector field over image space (millions of dimensions). Predicting ε (same size as the image) is equivalent but has a more concrete interpretation: "which noise vector was mixed in?" The MSE loss ||ε − ε̂||² is also simpler to optimize than the score matching loss.

Why DDPM Loss = Score Matching

The DDPM loss

// DDPM training objective (Ho et al. 2020, simplified): L_DDPM = E_{t,x_0,ε}[||ε − ε_θ(x_t, t)||²]   // This is just: for random t and random noise ε, // how well can the network predict the noise?

Equivalence to score matching — step by step

// Score matching loss (at noise level t): L_score = E[||s_θ(x_t, t) − ∇_{x_t} log p(x_t | x_0)||²]   // We know: ∇_{x_t} log p(x_t | x_0) = (x_0 − x_t) / (1−ᾱ_t)   // Substituting x_t = sqrt(ᾱ_t)×x_0 + sqrt(1−ᾱ_t)×ε: x_0 − x_t = −sqrt(1−ᾱ_t) × ε (approximately)   // So: target score = −ε / sqrt(1−ᾱ_t) // If s_θ = −ε_θ / sqrt(1−ᾱ_t), then: L_score = (1/(1−ᾱ_t)) × E[||ε − ε_θ(x_t, t)||²]   // L_score = L_DDPM × weight(t) // Same minimum, same gradient direction (weight is constant for fixed t) // Minimizing DDPM loss = minimizing score matching loss at each t

What this means intuitively

The diffusion model simultaneously learns the score function at every noise level. A single network, trained with MSE loss on noise prediction, implicitly learns:

  • At t=0 (clean): high-frequency score, sensitive to fine details
  • At t=500 (medium noise): medium-resolution score, captures shapes
  • At t=1000 (pure noise): smooth score, just says "move toward any real image"

Training with random t covers all noise levels simultaneously. This multi-scale score learning is why diffusion models generate such detailed, high-quality images.

Numerical example: score at two noise levels

// True pixel: x_0 = 0.8 (bright)   // At t=100 (ᾱ=0.90): x_100 = sqrt(0.90)×0.8 + sqrt(0.10)×ε // Say ε=0.2, so x_100 = 0.949×0.8 + 0.316×0.2 = 0.759+0.063 = 0.822 score(x_100) = −ε / sqrt(1−ᾱ_100) = −0.2 / sqrt(0.10) = −0.2 / 0.316 = −0.633 // Strong signal: points back toward the original (small noise → sharp score)   // At t=900 (ᾱ=0.001): x_900 = sqrt(0.001)×0.8 + sqrt(0.999)×ε // Say ε=1.2, so x_900 ≈ 0.025 + 0.9995×1.2 = 1.225 score(x_900) = −ε / sqrt(1−ᾱ_900) = −1.2 / sqrt(0.999) = −1.2 / 0.9995 = −1.201 // Weaker relative signal: high noise level → noisy score estimate

DDPM vs DDIM — Stochastic vs Deterministic Sampling

DDPM sampling — stochastic, 1000 steps

// DDPM reverse step (stochastic): x_{t-1} = (1/sqrt(α_t)) × (x_t − (β_t/sqrt(1−ᾱ_t)) × ε_θ(x_t, t)) + σ_t × z where z ~ N(0, I)   // σ_t = sqrt(β_t) — adds fresh noise at every step // This maintains the correct distribution at each t // But requires 1000 steps → slow (minutes per image)

Concrete DDPM step at t=10

// t=10, β_10=0.001, α_10=0.999, ᾱ_10≈0.990 // x_10 = 0.85 (nearly clean, small noise) // Network prediction: ε̂ = 0.12   mean = (1/sqrt(0.999)) × (0.85 − (0.001/sqrt(0.010)) × 0.12) = 1.0005 × (0.85 − (0.001/0.100) × 0.12) = 1.0005 × (0.85 − 0.01 × 0.12) = 1.0005 × (0.85 − 0.0012) = 1.0005 × 0.8488 = 0.8492   σ_10 = sqrt(β_10) = sqrt(0.001) = 0.0316   // Sample z ~ N(0,1), say z = 0.55: x_9 = 0.8492 + 0.0316 × 0.55 = 0.8492 + 0.0174 = 0.8666 // Note: added fresh noise z → stochastic output

DDIM — Denoising Diffusion Implicit Models (Song et al. 2020)

// DDIM reverse step (deterministic, σ=0): x_{t-1} = sqrt(ᾱ_{t-1}) × x̂_0 + sqrt(1−ᾱ_{t-1}) × ε_θ(x_t, t)   // Where: x̂_0 = (x_t − sqrt(1−ᾱ_t) × ε_θ) / sqrt(ᾱ_t)   // Key: NO z term (no fresh noise added) // Deterministic: same x_T always → same image

Same DDIM step at t=10

// Same inputs: x_10=0.85, ε̂=0.12, ᾱ_10=0.990, ᾱ_9=0.991   // Step 1: estimate clean image x̂_0 x̂_0 = (0.85 − sqrt(1−0.990) × 0.12) / sqrt(0.990) = (0.85 − sqrt(0.010) × 0.12) / 0.995 = (0.85 − 0.100 × 0.12) / 0.995 = (0.85 − 0.012) / 0.995 = 0.838 / 0.995 = 0.842   // Step 2: re-noise to level t-1=9 (ᾱ_9 ≈ 0.991) x_9 = sqrt(0.991) × 0.842 + sqrt(1−0.991) × 0.12 = 0.9955 × 0.842 + sqrt(0.009) × 0.12 = 0.838 + 0.0949 × 0.12 = 0.838 + 0.0114 = 0.849 // no randomness — same every time

The key differences

PropertyDDPMDDIM
Stochastic?Yes (adds z each step)No (deterministic)
Steps needed1000 (quality degrades with fewer)50–200 (quality preserved)
SpeedSlow (~30 sec/image)Fast (~2 sec/image)
Reproducible?No (different z each run)Yes (same x_T → same image)
Requires retraining?NoNo — same trained model
Used in practiceSometimes for qualityStable Diffusion default

DDIM uses the exact same trained model — it just changes the sampling procedure. No retraining needed. This is why it was a huge practical breakthrough.
