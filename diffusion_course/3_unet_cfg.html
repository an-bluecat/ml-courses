Diffusion Ch3: U-Net and Classifier-Free Guidance

The network architecture and how to steer generation

Why U-Net — Predicting Noise at Every Pixel

The task requires spatial precision

The network must predict the noise ε that was added to the image. Noise is a vector the same size as the image — every pixel has a noise value. A plain MLP or simple CNN would lose spatial structure. We need a network that processes the full image and produces a full-image output, understanding both global context and fine local details.

U-Net: encode down, decode up, skip connections

Encoder (downsample)

Repeatedly halve spatial resolution with conv+pool. Captures increasingly global context at smaller feature maps. Cheap computation at coarser scales.

Decoder (upsample)

Repeatedly double spatial resolution with upsampling+conv. Restores full resolution while using bottleneck's global context to guide reconstruction.

Skip connections concatenate encoder feature maps directly to the corresponding decoder layer — giving the decoder direct access to fine spatial details that the bottleneck cannot hold.

Concrete: 256×256 image through U-Net

// Input: noisy image x_t (256×256×3) + timestep t → predict ε (256×256×3)   // Encoder (downsampling path): E1: 256×256 → 128×128 (stride-2 conv, 64 channels) E2: 128×128 → 64×64 (stride-2 conv, 128 channels) E3: 64×64 → 32×32 (stride-2 conv, 256 channels) ← bottleneck   // Decoder (upsampling path) + skip connections: D1: 32×32 → 64×64 (upsample + concat E2 features) D2: 64×64 → 128×128 (upsample + concat E1 features) D3: 128×128 → 256×256 (upsample + 1×1 conv → ε̂)   // Output: ε̂ same shape as input image (256×256×3)

Why skip connections matter

The bottleneck (32×32) compresses the image 64×. It learns global structure — overall composition, broad shapes. But it loses fine-grained spatial detail. Skip connections bridge encoder and decoder: when the decoder reconstructs a 128×128 feature map, it concatenates the encoder's 128×128 features, which still contain the original edge positions, textures, and local patterns. Without skip connections, the predicted noise would be spatially blurry and misaligned.

Timestep Conditioning — Telling the Network What Noise Level

Why the network needs t

The same noisy image pixel value of 0.5 means very different things at t=1 (barely any noise added — almost certainly signal) versus t=999 (pure noise — value is nearly all noise). Without knowing t, the network cannot know which regime it's in. A single network trying to handle both would fail completely.

Without t (wrong)

Network sees x_t=0.5 at any noise level. At t=1, correct ε is tiny. At t=999, correct ε is ~0.5. Contradictory targets — network learns nothing useful.

With t (correct)

Network sees (x_t=0.5, t=1): predict tiny ε. Network sees (x_t=0.5, t=999): predict large ε. Different inputs → different outputs. Learnable.

Sinusoidal timestep embedding

The same technique as positional encoding in Transformers. Maps integer t to a continuous, unique vector that the network can process.

// Sinusoidal embedding for timestep t (dimension d=256): emb[2i] = sin(t / 10000^(2i/d)) // even dimensions: sin emb[2i+1] = cos(t / 10000^(2i/d)) // odd dimensions: cos   // Concrete: t=500, d=256, showing first 4 values (i=0,1): emb[0] = sin(500 / 10000^0) = sin(500) = 0.960 emb[1] = cos(500 / 10000^0) = cos(500) = -0.279 emb[2] = sin(500 / 10000^(2/256))= sin(500/1.018) = 0.922 emb[3] = cos(500 / 10000^(2/256))= cos(500/1.018) = -0.388 ... // 256 values total — each t gets a unique fingerprint

Injecting t into the U-Net

// Steps to condition the network on t: 1. t → sinusoidal embedding → 256-dim vector 2. Pass through small MLP: Linear(256→512) → SiLU → Linear(512→512) 3. At each U-Net level, project t_emb to channel size and ADD to feature maps   // Example: at encoder level E2 with 128 channels: t_proj = Linear(512 → 128)(t_emb) // [128] features = features + t_proj[None, :, None, None] // broadcast over spatial dims   // Every layer at every scale knows the exact noise level — crucial for quality

Why sinusoidal, not just t as a scalar?

A single scalar t (or a learned embedding) works, but sinusoidal embeddings have nice properties: (1) Different frequencies encode information at different scales of t, (2) Nearby timesteps get similar embeddings, aiding generalization, (3) No learned parameters needed — works out of the box. The different frequency terms let the network easily distinguish nearby timesteps (via high-frequency components) and far-apart ones (via low-frequency components).

Text Conditioning — Steering Generation with Language

The goal: "a cat on a mat" → that specific image

We want the noise prediction to be guided by a text description, so each denoising step moves toward images matching the text. The mechanism: encode the text to embeddings, then let the image features attend to the text embeddings at each U-Net decoder layer via cross-attention.

Step 1: text encoder

// Use a pretrained text encoder (CLIP or T5): text = "a cat on a mat" tokens = tokenize(text) // ["a", "cat", "on", "a", "mat"] → ids text_emb = TextEncoder(tokens) // shape: [5 tokens × 768 dims]   // Each token gets a 768-dim vector capturing its semantic meaning // "cat" embedding is near "kitten", "feline" — far from "dog" // CLIP encoder: trained on 400M (image, text) pairs — rich semantics

Step 2: cross-attention in the decoder

// Each U-Net decoder block has two attention operations:   // (A) Self-attention: image features attend to each other Q = W_Q × img_features // [H×W × d] K = W_K × img_features // [H×W × d] V = W_V × img_features // [H×W × d] out = softmax(QKᵀ/√d) × V // image regions attend to each other   // (B) Cross-attention: image features attend to text tokens Q = W_Q × img_features // queries come from the image K = W_K × text_emb // keys come from text V = W_V × text_emb // values come from text out = softmax(QKᵀ/√d) × V // image pixels "read" relevant text tokens

How cross-attention guides generation

A spatial region of the image (e.g., the area that will become a cat) learns to attend strongly to the "cat" token in the text embedding. This pulls that region's feature toward "cat-like" patterns during denoising. The "mat" token influences the bottom region. Over 1000 denoising steps, the cross-attention continuously redirects noise removal toward the described content.

Attention typeQK, VEffect
Self-attentionImage featuresImage featuresSpatial coherence, long-range consistency
Cross-attentionImage featuresText embeddingsSemantic guidance from the text prompt

Why decoder layers only?

The encoder downsamples aggressively — at the bottleneck, spatial resolution is very low. Cross-attention in the decoder makes sense because that's where the model is actively reconstructing spatial detail at multiple resolutions (64×64, 128×128, 256×256). Text tokens inform "what should the high-res features look like here?" at each scale.

Classifier-Free Guidance — Amplifying the Text Signal

The problem: conditioned generation is too weak

With text conditioning alone, the model hedges. It generates images that are plausible given the data distribution AND loosely match the text. The result: images that look realistic but don't strongly match the prompt. We want to amplify how much the text drives the output.

The CFG training trick

// During training: randomly drop the text condition with prob 0.9: train normally with text → ε_θ(x_t, t, c) // conditional with prob 0.1: replace text with null embedding ∅ → ε_θ(x_t, t, ∅) // unconditional   // After training, the same network can predict both: // - ε_cond: noise prediction given the text prompt // - ε_uncond: noise prediction with no text (generic image direction)

The CFG inference formula

// At each denoising step, run the network TWICE: ε_uncond = ε_θ(x_t, t, ∅) // unconditional prediction ε_cond = ε_θ(x_t, t, c) // conditional prediction (with text c)   // Combine with guidance scale w: ε_guided = ε_uncond + w × (ε_cond − ε_uncond)   // Interpretation: start at unconditional, then move FURTHER toward // conditional than the raw prediction — extrapolate past ε_cond

Concrete: w=7.5 step-by-step

// Say at one denoising step, for a single pixel (showing 3 dims): ε_uncond = [0.1, 0.2, -0.3] // generic "any image" direction ε_cond = [0.4, 0.8, 0.1] // "a cat on a mat" direction   // Difference (how much text changes the prediction): Δε = [0.3, 0.6, 0.4] // ε_cond − ε_uncond   // Scale by w=7.5: w × Δε = [2.25, 4.5, 3.0] // amplified text direction   // Final guided prediction: ε_guided = [0.1+2.25, 0.2+4.5, -0.3+3.0] = [2.35, 4.7, 2.7]   // The text direction is 7.5× stronger than the raw conditional signal

Guidance scale tradeoffs

w valueEffectUse case
w = 1No amplification (ε_guided = ε_cond)Raw conditional generation
w = 3–5Mild guidance — diverse, somewhat prompt-adherentCreative exploration
w = 7.5Strong guidance — vivid, prompt-adherent, less diverseStable Diffusion default
w = 15+Oversaturated, artifacts, less realisticNot recommended

CFG costs 2× compute per step (two network evaluations), but is considered essential for practical quality. It's used in virtually every production diffusion system.

Latent Diffusion — Running Diffusion in Compressed Space

The compute problem with pixel-space diffusion

Running DDPM on a 512×512 image requires a U-Net that operates on 512×512 feature maps. Each denoising step is expensive — and you need 1000 of them. Generating a single image takes minutes. Latent diffusion solves this by running the entire diffusion process in a much smaller compressed space.

The three-stage pipeline

// Stage 1: Compress with VAE encoder (done once per training image) image: 512×512×3 pixels VAE encoder: 512×512×3 → 64×64×4 latent // 8× spatial downsampling → 64× fewer elements (512²×3 vs 64²×4)   // Stage 2: Run diffusion in latent space (the expensive part) diffusion: 1000 denoising steps on 64×64×4 latent // U-Net operates on 64×64, NOT 512×512 → dramatically cheaper   // Stage 3: Decode with VAE decoder (done once at inference) VAE decoder: 64×64×4 → 512×512×3 image // Single forward pass — fast

Compute comparison: pixel vs latent space

// Cost of one U-Net forward pass (approximate FLOPs): Pixel DDPM on 512×512: ~1.2 GFLOP per denoising step Latent on 64×64: ~0.019 GFLOP per denoising step Speedup per step: ~63× faster   // Total inference (1000 steps): Pixel DDPM: 1000 × 1.2 = 1200 GFLOP Latent: 1000 × 0.019 = 19 GFLOP (+tiny VAE overhead) // Stable Diffusion 1.5 generates 512×512 in ~2 sec on a GPU // Pixel-space equivalent would take ~2 minutes

Why doesn't the VAE bottleneck hurt quality?

A naive compression (e.g., JPEG artifacts) would destroy fine details. The VAE is trained with a perceptual loss: it learns to preserve features that matter visually — edges, textures, semantic regions — rather than minimizing per-pixel error. The 64×64×4 latent space is high enough capacity to represent a 512×512 image's perceptually important content.

Diffusion then operates on this perceptually-aligned latent, so each denoising step directly shapes meaningful image features, not individual pixels. The quality is preserved because the VAE's encoding/decoding is deterministic and high-fidelity.
