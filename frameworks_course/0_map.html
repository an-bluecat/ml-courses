The Map

NumPy · PyTorch · TensorFlow · JAX — four answers to one question. Read this first; it's the spine of the whole course.

Everything starts from one problem: Python is too slow for math

Measure it — a dot product of 1,000,000 numbers

s = 0.0 for i in range(1_000_000): # plain Python s += a[i] * b[i] s = a @ b # numpy
output — real run, Apple M3, single run pure Python loop: 97.0 ms result 250031.5146 numpy a @ b : 0.844 ms result 250031.5146 ratio: 115x

Same numbers, same answer, 115× apart. The Python loop is slow because every single a[i] * b[i] goes through the interpreter: unbox the object, look up __mul__, allocate a result object, repeat — a million times. NumPy runs one loop written in C that multiplies raw doubles sitting next to each other in memory.

So the real question every framework answers

Where does the fast loop live — and what is Python's job?

NumPy (2005)Python calls one C loop per operation, immediately. Python IS the program — it runs every step.
TensorFlow 1 (2015)Python only describes a graph of operations; a C++ runtime executes it later, anywhere. Python is a metaprogram.
PyTorch (2017)Like NumPy — eager C/CUDA calls — but each call also records itself onto a tape, so gradients come free. Python is the program again.
JAX (2018)Python runs once while being traced; a compiler (XLA) rewrites and runs the math after that. Python generates the program.

Hold onto this one sentence per framework. Almost every difference you'll meet in this course — debugging pain, deployment stories, weird error messages, why JAX bans x[0] = 5 — falls out of these four choices.

The second question (deep learning only)

Training needs ∂loss/∂w for every weight. NumPy makes you derive it by hand. The other three compute it for you — but they disagree about where the gradient machinery lives: on the tensors (PyTorch), in a context manager (TensorFlow), or as a function transformation (JAX). That's Chapter 6.

Thirty years, four lineages — who begat whom

Solid dots: releases. Dashed arrows: ideas flowing — NumPy's array API into all three frameworks; Chainer's define-by-run and autograd's differentiate-plain-code into PyTorch; TensorFlow's XLA compiler into JAX.

One-line character sketches

FrameworkBuilt by / forIts bet
NumPyScientists unifying a fractured community (Travis Oliphant, 2005)One shared array type is worth more than any single feature
TensorFlowGoogle, to train & ship models on anything from datacenters to phonesDescribe the program as a graph; the graph is portable and optimizable

Each new system is a reaction to the previous one's pain: TF reacted to "research code can't be shipped," PyTorch to "graphs can't be debugged," JAX to "eager code can't be transformed."

The same computation in all four — look before we explain

The running example for the whole course

One weight w = 2.0, one input x = 3.0, one target t = 10.0:

pred = w * x # 6.0 loss = (pred - t)**2 # (-4)^2 = 16.0 # dloss/dw = 2·(pred-t)·x = 2·(-4)·3 = -24.0

Same numbers as the PyTorch Mechanics course — if you traced −24 by hand there, every framework below must reproduce it.

NumPy — you derive the gradient yourself

w, x, t = 2.0, 3.0, 10.0 loss = (w*x - t)**2 grad = 2*(w*x - t)*x # the chain rule, by hand

PyTorch — the tensors record a tape; grad lands in .grad

w = torch.tensor(2.0, requires_grad=True) loss = (w * 3.0 - 10.0)**2 loss.backward() # walks the recorded graph w.grad # the gradient WAITS IN A MAILBOX on w

TensorFlow 2 — you open a tape context; grad is returned

w = tf.Variable(2.0) with tf.GradientTape() as tape: # recording ON while inside loss = (w * 3.0 - 10.0)**2 grad = tape.gradient(loss, w) # returned to you

JAX — grad transforms the function itself

f = lambda w: (w * 3.0 - 10.0)**2 jax.grad(f) # a NEW FUNCTION that computes df/dw jax.grad(f)(2.0) # call it like any function
output — real run, all four numpy loss 16.0 grad -24.0 (derived by hand) torch loss 16.0 grad -24.0 tf loss 16.0 grad -24.0 jax loss 16.0 grad -24.0

Same math — but notice what moved

And a promise: in Chapter 9 we train the same network in all four with identical initial weights, and the loss curves match to six decimal places. Different dialects — same language.
