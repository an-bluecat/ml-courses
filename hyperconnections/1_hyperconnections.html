HyperConnections

What if one residual stream isn't enough?

The Residual Stream — One Vector Does Everything

What the residual stream actually is

Every transformer layer reads from a single vector and writes back to it. That vector is the "residual stream." After L layers, it's the sum of all the layer outputs.

// Standard residual transformer (single stream x):   x = embedding("the cat sat") // [1.0, 0.5] (2-dim, for illustration)   // Layer 1: attention + MLP f1_out = layer1(x) // = [0.30, 0.30] x = x + f1_out // = [1.30, 0.80] ← everything in one vector   // Layer 2: f2_out = layer2(x) // = [0.20, 0.20] x = x + f2_out // = [1.50, 1.00] ← final state

The problem: one stream, no lanes

Every layer writes to the same vector. Layer 1's features are mixed with layer 2's features. There's no way for early-layer syntactic info to "stay out of the way" of late-layer semantic info — they live in the same space and interfere.

It works in practice (transformers are very good), but the single stream is a bottleneck. All tasks — syntax, semantics, world knowledge, positional info — compete for the same 4096 dimensions.

// Analogy: a single whiteboard shared by 96 people. // Each person reads it, writes on it, and passes it to the next. // Late writers can accidentally erase early writers.

What HyperConnections proposes

Instead of one residual stream, keep n streams in parallel. Each layer reads a weighted combination of all streams, then writes its output back to the streams according to learned weights.

n=1 gives the standard residual connection. n=2 or n=4 gives multiple "lanes" of information.

Two Streams — A Concrete Walkthrough

Setup: n=2 streams, 2 layers

We start with stream 0 (s0) initialized from the embedding. Stream 1 (s1) starts at zero.

// Initial state (after embedding): s0 = [1.0, 0.5] // the main stream s1 = [0.0, 0.0] // second stream, empty   // Each layer has two learned weight vectors: β (beta) = how much each stream contributes to the layer INPUT α (alpha) = how much the layer OUTPUT flows back into each stream

Layer 1: beta=[0.8, 0.2], alpha=[0.6, 0.4]

// Combine streams to get the layer input: input = β[0]·s0 + β[1]·s1 = 0.8·[1.0, 0.5] + 0.2·[0.0, 0.0] = [0.80, 0.40]   // Run the transformer layer: f1_out = layer1(input) = [0.30, 0.30]   // Distribute the output back to each stream: s0 ← s0 + α[0]·f1_out = [1.0, 0.5] + 0.6·[0.30, 0.30] = [1.18, 0.68] s1 ← s1 + α[1]·f1_out = [0.0, 0.0] + 0.4·[0.30, 0.30] = [0.12, 0.12]

Layer 2: beta=[0.5, 0.5], alpha=[0.3, 0.7]

// Now layer 2 sees BOTH streams: input = 0.5·[1.18, 0.68] + 0.5·[0.12, 0.12] = [0.59, 0.34] + [0.06, 0.06] = [0.65, 0.40]   f2_out = layer2(input) = [0.20, 0.20]   // Distribute — this time mostly to stream 1: s0 ← [1.18, 0.68] + 0.3·[0.20, 0.20] = [1.24, 0.74] s1 ← [0.12, 0.12] + 0.7·[0.20, 0.20] = [0.26, 0.26]

Final output: pool across streams

// One common choice: read out stream 0 only output = s0 = [1.24, 0.74]   // Or sum both streams: output = s0 + s1 = [1.24+0.26, 0.74+0.26] = [1.50, 1.00]   // What changed vs standard residual? // s1 has isolated the "layer 1 update that was distributed to stream 1" // s0 carries the main signal; s1 is a separate lane

The Formula — and How Residual Is a Special Case

General HyperConnections update (n streams)

// State: S ∈ R^{n × d} (n streams, each d-dimensional)   // For each layer l with weights β_l ∈ R^n and α_l ∈ R^n:   // Step 1: aggregate streams into one input vector input_l = Σ_k β_l[k] · S[k] // weighted sum over n streams   // Step 2: run the transformer sublayer (attention or MLP) delta_l = f_l(input_l) // same transformer layer as before   // Step 3: distribute output back to all streams S[k] ← S[k] + α_l[k] · delta_l // for each stream k   // β and α are learned. They are either fixed (static) or // computed from the input (dynamic).

n=1 gives exact standard residual

// With n=1, there is only one stream S[0] = x and one weight each: β_l = [1.0] // must sum to 1, so it's just 1.0 α_l = [1.0] // one stream, full update   // input_l = 1.0 · x = x // delta_l = f_l(x) // x ← x + 1.0 · delta_l = x + f_l(x)   // That's exactly x = x + f(x). Standard residual.

Static vs Dynamic — Fixed Weights or Input-Dependent?

Static HyperConnections

β and α are learned parameters — one fixed value per layer, shared across all tokens and all time steps.

// Layer 3 always has the same weights: β_3 = [0.8, 0.2] // fixed after training α_3 = [0.3, 0.7] // fixed after training   // Same for every token in every batch. // Cost: 2·n floats per layer — essentially free.

Dynamic HyperConnections

β and α are computed from the current stream state — a small learned projection that looks at the input and decides how to route.

// Layer 3 computes per-token weights: β_3 = softmax(W_β · S[0]) // depends on token α_3 = softmax(W_α · S[0]) // W_β, W_α ∈ R^{n × d}   // "cat" might route differently than "the" // Cost: n·d extra params per layer — small.

Concrete dynamic example: same layer, two tokens

// W_β (1×2 matrix): [-0.5, +1.2] (learned projection → n=2 logits → softmax)   // Token "cat", s0=[1.0, 0.5]: logits = W_β · [1.0, 0.5]ᵀ = -0.5·1.0 + 1.2·0.5 = 0.10 // (this is n=1 logit; for n=2 we'd have two projections)   // For simplicity let's say W_β produces logits [0.10, -0.10]: β_cat = softmax([0.10, -0.10]) = [0.55, 0.45] // nearly equal   // Token "the", s0=[0.1, 0.1] (very flat, function word): // W_β produces logits [-0.8, 0.8]: β_the = softmax([-0.8, 0.8]) = [0.18, 0.82] // mostly stream 1   // "cat" routes through stream 0; "the" routes mostly through stream 1. // The streams can specialize per token type.

Why HyperConnections Beat Residual Connections

1. Better gradient flow

In a standard residual network, the gradient from the output flows to every previous layer through the residual path. But the layer outputs can still interfere with each other in the single stream.

// Standard residual: gradient of loss w.r.t. layer 1 output: ∂L/∂f1 = ∂L/∂x_final · 1 // chain rule through all additions   // HyperConnections: gradient also flows through α weights. // α controls how much of f1_out went to each stream. // If α=[0.6, 0.4], then 60% of f1's output went to s0, 40% to s1. // Gradient arrives through both paths — more signal to layer 1.   // Multiple gradient paths = more stable training at large depth.

2. Stream specialization

With training, different streams tend to carry different kinds of information. This is an emergent behavior — the model learns to keep streams separated because it helps minimize loss.

// Hypothetical learned pattern (from analyzing α values): // Layers 1-8: α = [0.9, 0.1] → most output goes to stream 0 // Layers 9-16: α = [0.1, 0.9] → most output goes to stream 1 // Layers 17-24: β = [0.5, 0.5] → reads both equally for final task   // Stream 0 becomes: "early feature extraction" // Stream 1 becomes: "high-level reasoning" // No one designed this — it fell out of training.

3. Long-range information preservation

In a standard transformer, information written by layer 1 gets "diluted" by 95 more layers writing to the same vector. With multiple streams, a layer can write to a stream that later layers rarely touch — essentially archiving its output.

// Stream 0: main trunk — every layer reads and writes heavily // Stream 1: archive — layer 2 writes to it; only the final layer reads it   // Layer 2's output in stream 1 survives 94 layers untouched. // In standard residual, it would have been diluted by all 94 subsequent writes.
