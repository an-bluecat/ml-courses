Chapter 1: Bits and Codes

How to send messages with 0s and 1s — and why some codes are better than others

Based on Visual Information Theory — colah's blog

You Need to Send a Message

Setup

Bob talks about 4 animals: dog, cat, fish, bird. He sends you one animal name at a time over a wire. The wire can only carry 0s and 1s. You need a code to translate animals into binary.

Simplest code: give every animal the same length

4 animals → need 2 bits to tell them apart (2 bits = 4 combinations):

AnimalCodeLength
dog002 bits
cat012 bits
fish102 bits
bird112 bits

Every message costs 2 bits. Simple. But can we do better?

Short Codes for Common Words

A better code

AnimalProbabilityCodeLength
dog1/201 bit
cat1/4102 bits
fish1/81103 bits

"dog" = just 0. One bit! Half the cost of the fixed code.

Does it actually save bits? Let's count.

Say Bob sends 800 messages following his usual distribution:

// Fixed-length code (2 bits each):
800 messages × 2 bits = 1600 bits
 
// Variable-length code:
400 dogs × 1 bit = 400
200 cats × 2 bits = 400
100 fish × 3 bits = 300
100 birds × 3 bits = 300
Total = 1400 bits
 
Saved 200 bits! Average = 1400/800 = 1.75 bits/message

Average message length, as a formula

// Weight each codeword length by how often it's used:
average = (1/2)×1 + (1/4)×2 + (1/8)×3 + (1/8)×3
       = 0.5 + 0.5 + 0.375 + 0.375
       = 1.75 bits

This 1.75 is going to become entropy in Chapter 2. But first we need to understand why these particular code lengths are optimal.

Every Codeword Has a Cost

Why can't every animal have a 1-bit code?

If dog = 0 and cat = 1, you've used up all possible 1-bit strings. There's no room left for fish or bird. Short codes are expensive — they use up code space.

The cost of a codeword of length L

Think of code space as a pie of size 1. A codeword of length L eats a slice of size:

cost = 1 / 2L
Length LCodeword exampleCost = 1/2LFraction of code space used
1 bit01/250% of all possible codes
2 bits101/425% of all possible codes
3 bits1101/812.5%
3 bits1111/812.5%
Total1/2 + 1/4 + 1/8 + 1/8 = 1.0 (pie is full!)

The costs must sum to ≤ 1. You can't overspend your code space.

Notice something?

AnimalProbabilityCode cost (1/2L)
dog1/21/2

The cost of each codeword equals the probability of the animal. That's the optimal code: spend code space in proportion to how often each symbol appears.

From Cost to Length: The Inversion

We know: cost = 1/2L

Question: if I decide to spend a certain cost, what length codeword does that buy me?

Solve for L, step by step

// Start with:
cost = 1 / 2L
 
// Flip both sides:
2L = 1 / cost
 
// Take log base 2 of both sides:
L = log₂(1 / cost)

Verify with actual numbers

// dog: cost = 1/2
L = log₂(1 / (1/2)) = log₂(2) = 1 bit ✔
 
// cat: cost = 1/4
L = log₂(1 / (1/4)) = log₂(4) = 2 bits ✔
 
// fish: cost = 1/8
L = log₂(1 / (1/8)) = log₂(8) = 3 bits ✔

Every number checks out against our code table from page 1.

Now the key step: plug in p(x) for cost

We showed that the optimal cost for animal x equals its probability p(x). So:

// Optimal codeword length for symbol x:
L(x) = log₂(1 / p(x))
 
// Verify:
dog: L = log₂(1 / (1/2)) = log₂(2) = 1 bit ✔
cat: L = log₂(1 / (1/4)) = log₂(4) = 2 bits ✔
fish: L = log₂(1 / (1/8)) = log₂(8) = 3 bits ✔
bird: L = log₂(1 / (1/8)) = log₂(8) = 3 bits ✔

This formula is the bridge to entropy. Optimal length = log₂(1/p(x)). Chapter 2 just averages this.
