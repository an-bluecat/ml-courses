Chapter 2: Entropy

The minimum average bits to communicate — and a measure of uncertainty

Entropy = Average Optimal Codeword Length

From Chapter 1 we have:

Animalp(x)Optimal length = log₂(1/p(x))
dog1/21 bit
cat1/42 bits
fish1/83 bits
bird1/83 bits

Entropy is just the weighted average of these lengths:

The calculation

H = p(dog) × L(dog) + p(cat) × L(cat) + p(fish) × L(fish) + p(bird) × L(bird)
 
  = (1/2)×1 + (1/4)×2 + (1/8)×3 + (1/8)×3
  = 0.5   + 0.5   + 0.375 + 0.375
  = 1.75 bits

The formula

// Replace L(x) with log₂(1/p(x)):
H(p) = Σ p(x) · log₂(1 / p(x))
 
// Equivalent form (just algebra: log(1/a) = -log(a)):
H(p) = −Σ p(x) · log₂(p(x))
 
// Both are the same number. The first is more intuitive.

That's entropy. The minimum average bits per message, using the best possible code.

Entropy Measures Uncertainty

Three distributions, three entropies

DistributiondogcatfishbirdEntropy
Certain (always dog)1.00000 bits
Bob's mix0.50.250.1250.1251.75 bits
Uniform (all equal)0.250.250.250.252.0 bits

Verify the extremes with numbers

// Certain: p(dog)=1, rest=0
H = 1 × log₂(1/1) + 0 + 0 + 0
  = 1 × 0 = 0 bits
// "I know exactly what you'll say — no message needed."
 
// Uniform: p(x)=1/4 for all x
H = 4 × (1/4) × log₂(1/(1/4))
  = 4 × (1/4) × 2
  = 2 bits
// "No idea what you'll say — need the full 2-bit code."

The pattern

More concentrated probability → shorter codes → lower entropy → less uncertainty

More spread out probability → longer codes → higher entropy → more uncertainty

Entropy is simultaneously:

  • The minimum average bits to communicate an event
  • A measure of how uncertain/surprised you should expect to be

One-hot labels have zero entropy

This will matter in Chapter 3. In neural network training, the true label is one-hot: [1, 0, 0, 0] = "this image is definitely a dog." That's a certain distribution. Entropy = 0.

This means: H(true label) = 0. Remember this for later.
