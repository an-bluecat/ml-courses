Chapter 3: Cross-Entropy

What happens when you use the wrong code — and why this is the loss function in ML

Using Someone Else's Code

Bob (dog person) and Alice (cat person)

AnimalBob's distribution q(x)Alice's distribution p(x)
dog1/21/8
cat1/41/2
fish1/81/4
bird1/81/8

Bob designed his code for his distribution (dogs are common). Alice talks differently (cats are common).

Bob's code (optimized for Bob, not Alice)

AnimalBob's codeLength
dog01 bit (short — Bob says dog a lot)
cat102 bits
fish1103 bits
bird1113 bits

Now Alice uses Bob's code. What happens?

Alice says "cat" half the time, but Bob's code gives cat 2 bits (not 1). She's wasting bits.

// Alice's messages, Bob's codeword lengths:
avg length = p(dog)×L(dog) + p(cat)×L(cat) + p(fish)×L(fish) + p(bird)×L(bird)
           = (1/8)×1 + (1/2)×2 + (1/4)×3 + (1/8)×3
           = 0.125 + 1.0 + 0.75 + 0.375
           = 2.25 bits
 
// Alice's entropy (if she used her own optimal code):
H(Alice) = (1/8)×3 + (1/2)×1 + (1/4)×2 + (1/8)×3 = 1.75 bits
 
// Waste from using the wrong code:
2.25 - 1.75 = 0.5 extra bits per message

Alice could communicate in 1.75 bits with her own code. Using Bob's code costs 2.25. The 0.5 extra bits are pure waste.

Cross-Entropy Formula

What we just calculated has a name

Cross-entropy H(p, q) = average message length when:

  • The real distribution is p (Alice's — what actually happens)
  • The code is designed for q (Bob's — what we assumed would happen)

Building the formula from what we already know

// Entropy uses: p(x) weights, p(x) code lengths
H(p) = Σ p(x) · log₂(1/p(x))
 
// Cross-entropy uses: p(x) weights, q(x) code lengths
H(p, q) = Σ p(x) · log₂(1/q(x))
 
// Same structure. Just swap which distribution designs the code.
// The weights are always p (what really happens).
// The code lengths come from q (what we assumed).

Key property: cross-entropy ≥ entropy. Always.

H(Alice, Bob's code) = 2.25 ≥ H(Alice) = 1.75
 
// The wrong code always costs at least as much as the right one.
// Equal only when p = q (you used the right code).

Cross-Entropy in Neural Network Training

Map it to ML

Info theoryML training
p(x) = true distributionThe label: [1, 0, 0] = "this is a dog"
q(x) = assumed distributionModel output: [0.5, 0.3, 0.2]
Cross-entropy H(p, q)The loss function you minimize

Concrete example: image classifier

// True label (one-hot): this image is a dog
p = [1, 0, 0] // dog=1, cat=0, fish=0
 
// Model output (softmax probabilities):
q = [0.7, 0.2, 0.1] // model thinks 70% dog, 20% cat, 10% fish
 
// Cross-entropy:
H(p, q) = 1 × log₂(1/0.7) + 0 × log₂(1/0.2) + 0 × log₂(1/0.1)
        = 1 × 0.515 + 0 + 0
        = 0.515 bits
 
// Only the correct class survives! Because p is one-hot,
// every term with p(x)=0 vanishes. You're left with:
H(p, q) = −log(q(correct class)) = −log(0.7) = 0.515

What happens as the model gets better?

// Bad model: q(dog) = 0.1
loss = −log(0.1) = 3.32 bits // high loss!
 
// Decent model: q(dog) = 0.7
loss = −log(0.7) = 0.51 bits
 
// Great model: q(dog) = 0.99
loss = −log(0.99) = 0.014 bits // almost zero!
 
// Perfect model: q(dog) = 1.0
loss = −log(1.0) = 0 bits // loss = 0, model is perfect

Training a neural network = minimizing cross-entropy = making the model's code match reality.

One-hot labels make cross-entropy = KL divergence

Remember: H(one-hot) = 0. And KL = cross-entropy minus entropy. So:

KL(p ‖ q) = H(p, q) − H(p) = H(p, q) − 0 = H(p, q)

When the label is one-hot, cross-entropy loss IS KL divergence. Same number. This is why minimizing cross-entropy loss is equivalent to minimizing the distance between model and truth.
