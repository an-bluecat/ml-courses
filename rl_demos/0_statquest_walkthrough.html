Policy Gradient: The Fry Shop Problem

StatQuest walkthrough — every line of code explained, every number shown

Step 0: What Are We Actually Solving?

You're hungry. There are two fry shops:

🍟
Squatch's Fry Shack
Sometimes big, sometimes small servings
🍟
Norm's Fry Hut
Sometimes big, sometimes small servings

The hidden truth (you don't know this):

  • When you're not hungry (hunger=0) → Squatch gives small fries (good!), Norm gives big fries (bad, wasteful)
  • When you're very hungry (hunger=1) → Norm gives big fries (good!), Squatch gives small fries (bad, not enough)

The Goal

Build a neural network that takes hunger level as input and outputs which shop to visit.

Hunger
(0.0 to 1.0)
→
Neural Net
(1 weight + 1 bias)
→
P(Norm)
P(Squatch) = 1 - P(Norm)

Why Not Just Use Supervised Learning?

In supervised learning, you'd have a dataset like:

HungerCorrect Shop (Label)
0.0Squatch's
0.5???
1.0Norm's

But you don't have labels! Nobody tells you the right answer. You have to go there, order fries, and find out. That's why this is RL.

Step 1: Supervised Learning vs RL — The Key Difference

Supervised Learning

# You HAVE the answer
X = [cat_photo, dog_photo]
y = ["cat", "dog"]  # ← LABELS

# Training:
pred = model(X)
loss = cross_entropy(pred, y)
loss.backward()  # gradient always correct

Label tells you the right answer → gradient points the right way → done.

Reinforcement Learning

# You DON'T have the answer
hunger = 0.0
y = ???  # ← NO LABEL

# Training:
pred = model(hunger)
action = sample(pred) # pick randomly
reward = go_eat(action) # +1 or -1
# now what??

No label → you guess → go try it → reward tells you if guess was right or wrong.

The RL Trick (the whole idea)

1. Compute the gradient as if your guess was correct (pretend it's supervised)

2. Multiply gradient by the reward

3. If reward = +1 → gradient stays the same → push toward guess

4. If reward = -1 → gradient flips direction → push away from guess

# THE ENTIRE ALGORITHM IN 4 LINES:
gradient = compute_gradient(pred, guess)  # pretend guess is the label
gradient = gradient * reward              # reward confirms or flips it
bias = bias - lr * gradient              # standard gradient descent
# That's it. That's policy gradient.

Step 2: Forward Pass — Running the Network

The Network (tiny: 1 input, 1 weight, 1 bias, sigmoid)

# Our "neural network"
weight = 0.0   # starts at 0
bias   = 0.0   # starts at 0 ← THIS is what we're training

def sigmoid(x):
    return 1 / (1 + exp(-x))

def forward(hunger):
    x = hunger * weight + bias    # linear combination
    p_norm = sigmoid(x)           # squash to [0, 1]
    p_squatch = 1 - p_norm       # probabilities sum to 1
    return p_norm, p_squatch

Let's run it: hunger = 0.0

hunger = 0.0

# Step 1: linear combination
x = 0.0 * 0.0 + 0.0   # = 0.0

# Step 2: sigmoid
p_norm = 1 / (1 + exp(-0.0))
       = 1 / (1 + 1.0)
       = 1 / 2
       = 0.5             # 50% chance go to Norm's

# Step 3: complement
p_squatch = 1 - 0.5
          = 0.5          # 50% chance go to Squatch's

The network says: "I have no idea. 50/50."

Squatch 50%
Norm 50%

Now sample an action

# Pick a random number between 0 and 1
rand = 0.2   # (imagine this came from random.random())

# 0.2 < 0.5, so we land in Squatch's region
if rand < p_squatch:
    action = "squatch"   # ← we're going here
else:
    action = "norm"

# We chose Squatch's. But was it the right choice? We don't know yet.

Step 3: The "Guess" — Making a Fake Label

This is the weird part

We picked Squatch's randomly. Now we pretend that was the right answer.

We create a fake label: "the ideal probability for Squatch should be 1.0"

# We sampled: action = "squatch"
# Now PRETEND this is the correct label:
ideal_p_squatch = 1.0   # ← our GUESS (fake label)
actual_p_squatch = 0.5  # ← what the network said

# Gap: the network said 0.5 but "should" say 1.0
# We'll use this gap to compute a gradient.
# BUT the gradient might be WRONG if the guess is wrong.
# That's OK — the reward will fix it later.

Why does this work?

If guess was RIGHT
(Squatch really was the right choice)

reward = +1
gradient stays → push toward Squatch

If guess was WRONG
(Squatch was actually a bad choice)

reward = -1
gradient flips → push AWAY from Squatch

The guess can be wrong. The reward corrects it. You never need true labels.

Step 4: Cross Entropy — How Wrong Is the Network?

What is cross entropy?

It measures: "how surprised am I that the network gave probability P to the thing that happened?"

C = -log(Pchosen)

P(chosen action)-log(P)Meaning
1.0 (certain)0.0Not surprised at all
0.5 (coin flip)0.69Somewhat surprised
0.1 (unlikely)2.30Very surprised
0.014.60Shocked

Our calculation

import math

# We chose Squatch's, so we need P(Squatch)
p_squatch = 0.5

# Cross entropy: how far is the network from our guess?
C = -math.log(p_squatch)
  = -math.log(0.5)
  = -(-0.693)
  = 0.693

# Interpretation: the network gave 50% to our choice.
# If it believed us fully, C would be 0.
# We want to minimize C → push P(Squatch) toward 1.0

Rewrite in terms of P(Norm) — because that's the network's actual output

# Since P(Squatch) = 1 - P(Norm):
C_squatch = -math.log(1 - p_norm)
           = -math.log(1 - 0.5)
           = -math.log(0.5)
           = 0.693

# This form is what we'll take the derivative of.

Step 5: The Derivative — Chain Rule Step by Step

The chain of dependencies

Csquatch
-log(1-p_norm)
←
p_norm
sigmoid(x)
←
x
hunger*w + b
←
bias
what we train

dC/dBias = dC/dp_norm × dp_norm/dx × dx/dBias

Term 1: dC/dp_norm

# C_squatch = -log(1 - p_norm)
# Using chain rule: d/dp[-log(1-p)] = 1/(1-p)

dC_dp = 1 / (1 - p_norm)
      = 1 / (1 - 0.5)
      = 1 / 0.5
      = 2.0

Term 2: dp_norm/dx (sigmoid derivative)

# The famous sigmoid derivative: σ(x) * (1 - σ(x))

dp_dx = p_norm * (1 - p_norm)
      = 0.5 * (1 - 0.5)
      = 0.5 * 0.5
      = 0.25

Term 3: dx/dBias

# x = hunger * weight + bias
# dx/dBias = 1 (bias is just added, so derivative is 1)

dx_db = 1.0

Multiply all three together

dC_dBias = dC_dp * dp_dx * dx_db
         = 2.0  *  0.25  *  1.0
         = 0.5

# This derivative says: "to reduce C, shift bias to the LEFT"
# Which means: "make P(Squatch) bigger"
# Which means: "go to Squatch's more often"
#
# BUT WAIT — this is based on the GUESS that Squatch is right.
# What if the guess is wrong? That's where the reward comes in.

Step 6: The Reward — Confirm or Flip the Gradient

Go to Squatch's and order fries

We're not hungry (hunger = 0.0). Squatch gives us a small order.

Small fries when not hungry = exactly what we wanted!

# We went to Squatch's when not hungry.
# Squatch gave small fries → perfect for not being hungry!
reward = +1.0   # correct guess!

# If Squatch had given us HUGE fries (and we're not hungry):
# reward = -1.0  # wrong guess! too much food!

Multiply gradient by reward

gradient = 0.5       # from Step 5
reward   = 1.0       # correct guess

updated_gradient = gradient * reward = 0.5 * 1.0 = 0.5

# Reward = +1 → gradient unchanged → shift bias to the left
# This makes P(Squatch) bigger. Correct!

What if the reward had been -1?

gradient = 0.5
reward   = -1.0      # wrong guess!

updated_gradient = 0.5 * (-1.0) = -0.5

# FLIPPED! Now the gradient points RIGHT instead of LEFT.
# This makes P(Squatch) SMALLER. "Don't go there again!"

This is the entire insight of policy gradients

ScenarioGradientRewardUpdated GradientEffect
Right guess+0.5+1+0.5More Squatch
Wrong guess+0.5-1-0.5Less Squatch

The gradient assumes the guess is right. The reward confirms or corrects. No labels needed.

Step 7: The Update — Gradient Descent

Standard gradient descent, nothing new here

lr = 1.0              # learning rate
updated_gradient = 0.5 # from Step 6

# Step size
step_size = lr * updated_gradient
          = 1.0 * 0.5
          = 0.5

# New bias
bias_new = bias_old - step_size
         = 0.0 - 0.5
         = -0.5

Did it work? Run forward pass again with new bias:

# Same input: hunger = 0.0, weight = 0.0, NEW bias = -0.5
x = 0.0 * 0.0 + (-0.5) = -0.5
p_norm = sigmoid(-0.5) = 0.378
p_squatch = 1 - 0.378 = 0.622

Before update:

Squatch 50%
Norm 50%

After update:

Squatch 62%
Norm 38%

It worked! One update and the network already leans toward Squatch's when not hungry. After many more rounds (with different hunger levels), the bias converges to ~-10 and the network fully learns the right policy.

Step 8: Full Training — Watch It Learn

The complete algorithm, all at once

import math, random

weight = 0.0
bias   = 0.0
lr     = 1.0

for episode in range(200):
    hunger = random.random()                          # random hunger 0-1
    x = hunger * weight + bias                          # linear
    p_norm = 1 / (1 + math.exp(-x))                    # sigmoid

    # SAMPLE action
    went_to_norm = random.random() < p_norm

    # GET REWARD (the hidden truth)
    if went_to_norm:
        reward = +1 if hunger > 0.5 else -1          # Norm good when hungry
    else:
        reward = +1 if hunger <= 0.5 else -1         # Squatch good when not

    # COMPUTE GRADIENT (assuming our choice was correct)
    if went_to_norm:
        dC_db = -(1 - p_norm)                           # derivative for Norm
    else:
        dC_db = p_norm                                   # derivative for Squatch

    # REWARD CORRECTS THE GRADIENT
    dC_db = dC_db * reward                               # ← THE KEY LINE

    # UPDATE
    bias = bias - lr * dC_db
