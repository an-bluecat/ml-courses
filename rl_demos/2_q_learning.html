Chapter 2: Q-Learning

A 2x2 grid. Every single number traced. No interaction needed — just read.

The World: 2x2 Grid

Before you start: LLM post-training does NOT use Q-learning

PPO and GRPO (ch. 5, 7-8) are descendants of REINFORCE (ch. 3) — the policy-gradient family. No Q-table, no max Q(s') anywhere. Q-learning's home turf is games and robotics (DQN, Atari).

So why read this chapter? Two things get born here and reused everywhere:

• the vocabulary — target, error, gamma — which PPO's critic (ch. 4-5) uses unchanged;
• the idea of bootstrapping (substituting a stored guess for the real future). REINFORCE is defined by refusing to do this — you can't see why that matters until you've watched it work.

(0,0)-1🤖START (0,1)-1 empty
(1,0)-10🕳️PIT (1,1)+10🏆GOLD

Rules

Actions: up, down, left, right. If you walk into a wall, you bounce back to the same cell.

Every step costs -1. Even hitting a wall.

Pit (1,0): reward = -10, episode ends.

Gold (1,1): reward = +10, episode ends.

Who ends the episode? The environment. Each step the world returns three things: new_position, reward, done. When done=True (pit or gold), the episode is over. The robot doesn't choose to stop — the world forces it.

The Q-table: starts completely blank

4 numbers per cell (one per direction). All zero = "I have no idea."

Cell↑ up↓ down← left→ right
(0,0)0000
(0,1)0000

(Pit and Gold don't need Q-values — you never take actions from terminal cells.)

Settings we'll use throughout

alpha = 0.5   # learning rate — how fast to update
gamma = 0.9   # discount — future is worth 90% per step

Episode 1, Step 1

The robot is at (0,0). All Q-values are 0, so it picks randomly: RIGHT.

(0,0)🤖➡️ (0,1) 
(1,0)🕳️ (1,1)🏆

The world returns: new_position=(0,1), reward=-1, done=False

done=False means the episode continues. (0,1) is just empty floor.

Now update Q[(0,0), right]

Was: Q[(0,0), right] = 0
 
// What reward did I just get?
reward = -1
 
// What's the best Q at where I landed?
Q at (0,1) = [up=0, down=0, left=0, right=0]
best of those = 0
 
// Target = reward + gamma * best future
target = -1 + 0.9 × 0 = -1
 
// How wrong was the old value?
error = -1 − 0 = -1
 
// Nudge toward target
new Q = 0 + 0.5 × (-1) = -0.5

Q-table after this step

Cell↑↓←→
(0,0)000-0.5
(0,1)0000

One number changed. Robot knows: "going right from (0,0) costs about -0.5."

Episode 1, Step 2

Robot is at (0,1). All Q-values there are 0. Picks randomly: DOWN.

(0,0)  (0,1)🤖⬇️
(1,0)🕳️ (1,1)🏆

Wait — (0,1) going DOWN lands on (1,1) = GOLD!

The world returns: new_position=(1,1), reward=+10, done=True

done=True — the environment ended the episode. The robot didn't choose to stop. The world said "you found gold, game over."

Update Q[(0,1), down]

Was: Q[(0,1), down] = 0
 
reward = +10 // GOLD!
 
// done=True → there IS no next step → best future = 0
best future = 0 // game is over, no more moves
 
target = 10 + 0.9 × 0 = 10
error = 10 − 0 = +10
 
new Q = 0 + 0.5 × 10 = +5.0

Q-table after Episode 1

Cell↑↓←→
(0,0)000-0.5
(0,1)0+5.000

Two numbers changed. The robot now knows:

  • "Right from (0,0)" = -0.5 (meh, costs a step)
  • "Down from (0,1)" = +5.0 (great — that's the gold!)

Episode over. Robot resets to (0,0). But the Q-table keeps its values.

Episode 2, Step 1 — The Key Moment

This is where the magic happens

Robot is back at (0,0). Picks randomly: RIGHT again. Same move as before. But the Q-table at (0,1) is different now — it has the +5.0 from finding gold.

(0,0)🤖➡️ (0,1) 
(1,0)🕳️ (1,1)🏆

World returns: new_position=(0,1), reward=-1, done=False

Update Q[(0,0), right]

Was: Q[(0,0), right] = -0.5 // from episode 1
 
reward = -1
 
// What's the best Q at (0,1)?
Q at (0,1) = [up=0, down=+5.0, left=0, right=0]
best of those = +5.0 // ← the gold knowledge from episode 1!
 
target = -1 + 0.9 × 5.0 = -1 + 4.5 = +3.5
error = 3.5 − (-0.5) = +4.0
 
new Q = -0.5 + 0.5 × 4.0 = +1.5

Q-table now

Cell↑↓←→
(0,0)000+1.5
(0,1)0+5.000

This is reward propagation

The gold's +10 turned into +5.0 at (0,1) in episode 1.

Now that +5.0 turned into +1.5 at (0,0) — one cell further away.

The robot at (0,0) has never seen the gold. But it now knows: "going right is good (+1.5) because it leads somewhere promising."

🏆 GOLD  → +5.0 at (0,1) → +1.5 at (0,0)

Knowledge flows backward from the reward, one cell at a time.

Episode 5: Finding the Pit

Eventually the robot explores DOWN from (0,0)

(0,0)🤖⬇️ (0,1) 
(1,0)🕳️ (1,1)🏆

World returns: new_position=(1,0), reward=-10, done=True

done=True — environment killed the episode.

Update Q[(0,0), down]

Was: Q[(0,0), down] = 0
 
reward = -10
best future = 0 // done=True, no next step
 
target = -10 + 0.9 × 0 = -10
error = -10 − 0 = -10
 
new Q = 0 + 0.5 × (-10) = -5.0

Q-table now

Cell↑↓←→
(0,0)0-5.00+1.5
(0,1)0+5.000

Now (0,0) has two opinions:

  • Right = +1.5 (leads toward gold)
  • Down = -5.0 (leads to pit)

If the robot stops exploring and just picks the best Q-value, it will go RIGHT. That's the correct policy!

After ~50 Episodes: Converged

Every cell knows what to do

After repeating this process many times, the Q-values settle:

Cell↑↓←→
(0,0)-5.3-5.0-5.3+3.1 ★
(0,1)-0.5+7.5 ★+1.5-0.5

★ = best action in that cell (what the robot will pick)

The learned policy

(0,0)🤖➡️go RIGHT (0,1)⬇️go DOWN
(1,0)🕳️PIT (1,1)🏆GOLD

(0,0) → RIGHT → (0,1) → DOWN → GOLD. Shortest path. Avoids pit.

How it got there — the whole mechanism in one picture

🏆 Gold (+10)  →  Q[(0,1), down] learned +7.5
↓
That +7.5  →  Q[(0,0), right] learned +3.1   (= -1 + 0.9 × 7.5, roughly)
 
🕳️ Pit (-10)  →  Q[(0,0), down] learned -5.0

Reward flows backward from terminal cells, one hop at a time. That's Q-learning.

The Formula — You Already Did It

Q(s, a) ← Q(s, a) + α · [ r + γ · maxa' Q(s', a') − Q(s, a) ]

Every symbol = something you already saw

SymbolWhat it was in our examples
Q(s, a)The number in the table we were updating
α (alpha)0.5 — how big a step toward the target
r-1 (step), -10 (pit), or +10 (gold)
γ (gamma)0.9 — future is worth 90%
max Q(s', a')"best Q at the cell I landed in"
done = Truemax Q(s', a') = 0 — no future exists

That's it

Every "scary" formula term is just one of the numbers you watched change in the Q-table. The formula is a compressed way of writing the 5 lines of math you saw on every step.
