What Does the Network Look Like?

Our 2x2 grid from Chapter 2 — same world, new brain

The network replaces the Q-table. Instead of looking up values in a table, we feed the state in and get action probabilities out.

Input: "Where am I?" — one-hot encoded

The 2x2 grid has 4 cells. We represent the robot's position as 4 numbers: a 1 in the current cell, 0s elsewhere.

The network outputs one probability for each action. They sum to 1 (via softmax).

In code

import torch.nn as nn

policy_network = nn.Sequential(
    nn.Linear(4, 8),      # 4 inputs → 8 hidden  (4×8 = 32 weights)
    nn.ReLU(),              # activation function
    nn.Linear(8, 4),      # 8 hidden → 4 outputs (8×4 = 32 weights)
    nn.Softmax(dim=-1),   # turn raw scores into probabilities (sum to 1)
)

# Use it:
state = [1, 0, 0, 0]        # "I'm at (0,0)"
probs = policy_network(state) # → [0.05, 0.10, 0.05, 0.80]
action = sample(probs)        # → 3 (= right), picked randomly with 80% chance

For our tiny 2x2 grid the network is overkill (76 params vs 8). The point is that the same architecture works when the state is a 1000-dim vector or a camera image.

The REINFORCE Idea — No Math

Run a full episode. Then look back.

# Episode on our 2x2 grid:
# Step 0: at (0,0), went RIGHT    → reward -1
# Step 1: at (0,1), went DOWN     → reward +10  (gold!)
# Total return: -1 + 10 = +9 (great!)

The rule: make good episodes more likely, bad ones less

For each action in the episode:

If the return from this point was HIGH: increase the probability of this action

If the return from this point was LOW: decrease the probability

That's it. That's REINFORCE. Now how do you "increase the probability"?

Log probability = the knob you turn

# The network outputs probabilities like P(down) = 0.6
# log(0.6) = -0.51
#
# To make an action MORE likely: increase log P(action)
# To make an action LESS likely: decrease log P(action)
# Gradient of log P naturally does this — it's how softmax works.
#
# And to scale HOW MUCH to push: multiply by the return G
# Big G → big push.  Small G → small push.  Negative G → push the other way.

One Episode, Step by Step

1. Collect the full episode (no learning yet, just play)

saved = []  # we'll save everything for later

state = (0, 0)  # start
while not done:
    probs = network(state)   # e.g. [0.1, 0.6, 0.05, 0.25]
    action = sample(probs)   # randomly pick based on probs
    reward, next_state, done = env.step(action)

    saved.append( (state, action, reward) )
    state = next_state

# saved = [(s0,a0,r0), (s1,a1,r1), ..., (sT,aT,rT)]

2. Compute returns (walk backward through the episode)

# Rewards collected: [-1, -1, -1, -1, -1, +10]
# Return at step t = reward_t + gamma*reward_{t+1} + gamma^2*reward_{t+2} + ...
# Easier: walk backward

G = 0
returns = []
for r in reversed(rewards):
    G = r + gamma * G
    returns.insert(0, G)

# With gamma=0.99, rewards=[-1,-1,-1,-1,-1,+10]:
# Step 5: G = 10                          = 10.00
# Step 4: G = -1 + 0.99*10                =  8.90
# Step 3: G = -1 + 0.99*8.90              =  7.81
# Step 2: G = -1 + 0.99*7.81              =  6.73
# Step 1: G = -1 + 0.99*6.73              =  5.66
# Step 0: G = -1 + 0.99*5.66              =  4.60
#
# returns = [4.60, 5.66, 6.73, 7.81, 8.90, 10.00]
# ALL positive! Every step of this episode was good.

3. Compute the loss and update

loss = 0
for t in range(len(saved)):
    state, action, _ = saved[t]
    probs = network(state)

    loss += -log(probs[action]) * returns[t]
    #         ^^^^^^^^^^^^^^^^    ^^^^^^^^^^^
    #         "how surprised"     "how good was it"
    #
    # High return + low probability → BIG loss
    # "This action was great but I almost didn't pick it — fix that!"
    #
    # Low return + high probability → also pushes (negatively)
    # "This action was bad and I keep picking it — stop!"

loss.backward()   # compute gradients
optimizer.step()  # update the network

The fry shop was one step, one state. Here it's many steps, many states. But the core idea — "gradient * reward signal" — is exactly the same.

The big problem: NOISE

# Episode A: stumbles around, accidentally finds treasure  → return = +2
# Episode B: takes a direct path but treasure was 1 step away → return = +9
#
# REINFORCE says: push BOTH episodes' actions proportionally
# Episode A's random stumbling gets reinforced too!
#
# Even worse:
# Episode C: return = +5 (good? bad? no idea without context!)
# If average return is +3, then +5 is great.
# If average return is +8, then +5 is terrible.
# Raw G doesn't tell you which.
#
# This is the VARIANCE PROBLEM. Chapter 4 fixes it.
