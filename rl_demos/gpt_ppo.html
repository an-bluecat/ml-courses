A policy-update laboratory

PPO, without the magic trick.

Start with one ordinary actor update. Reuse the data. Watch it become stale. Only then earn the ratio, the one-sided brake, and the compact PPO formula.

The whole thesis

PPO lets you learn several times from one expensive batch while preventing any one stale advantage from earning unlimited policy change.

00

First, what problem are we solving?

Actor–Critic already knows how to improve a policy. PPO begins when we ask to learn more than once from the same expensive experience.

One fresh update
Ordinary policy gradient is enough.
loss = -log(prob_taken) * advantage
update(loss)
# discard batch; collect fresh data
The expensive part
Environment interaction.

A simulator, robot, game, or human-feedback pipeline may take far longer than another neural-network backward pass. Throwing the batch away after one update wastes useful evidence.

COLLECTRun the old policy
FREEZESave old probabilities and advantages
UPDATERead the batch once
REUSERead it for more epochs
BRAKEStop trusting samples that moved enough
The danger: the advantage is frozen, but the policy changes after every optimizer step. Reading “this action was good” repeatedly can push its probability much farther than that old evidence deserves.
01

The clipping idea, with one sample.

No ratio formula yet. Just compare the current probability with a boundary computed from the frozen old probability.

Positive advantage

The action did better than expected, so increase its probability—but stop giving this sample gradient after a 20% relative increase.

upper = old_prob * 1.2
active = current_prob < upper

Negative advantage

The action did worse than expected, so decrease its probability—but stop giving this sample gradient after a 20% relative decrease.

lower = old_prob * 0.8
active = current_prob > lower

02

One batch, many independent brakes.

A batch is just the one-sample rule performed element by element. Some samples remain active while others stop contributing policy gradient.

The readable teaching implementation

upper = old_probs * 1.2
lower = old_probs * 0.8

active = torch.where(
    advantages > 0,
    new_probs < upper,    # good action: not increased too far
    new_probs > lower,    # bad action: not decreased too far
)

teaching_loss = -(
    torch.log(new_probs) * advantages * active
).mean()

Why not stop the whole epoch? Sample C is finished, but A, B, and D still carry useful gradients. The brake must be per sample, not one global flag.
Checkpoint: why the brake must be one-sided

Positive advantage + probability fell:

That is the wrong direction. Even if it fell below 0.8×, do not mask it; its gradient needs to pull it back up.

Negative advantage + probability rose:

Also the wrong direction. Even if it rose above 1.2×, keep its gradient so it can be pushed back down.

03

Why the ratio exists at all.

The mask explains the brake. The ratio solves a different problem: after an update, the saved batch still has the old policy’s action frequencies.

Frozen batch
40 A samples, 60 B samples.

Those counts came from an old policy with P(A)=0.40. They do not magically change after the network says P(A)=0.46.

Current policy
Should behave like 46 A, 54 B.

If we could afford to recollect immediately, this is the action mixture we would expect in another 100 samples.

The ratio turns old counts into new effective counts

ratio = new probability / old probability

A400.400.461.1540 × 1.15 = 46
B600.600.540.9060 × 0.90 = 54

A complete advantage check

Let old-policy advantages be A(A)=+3 and A(B)=−2. Under the old policy they average to zero:

0.40(3) + 0.60(−2) = 0

The candidate policy shifts weight toward A:

0.46(3) + 0.54(−2) = +0.30

Using old samples plus ratios recovers exactly the same candidate-policy score:

0.40(1.15)(3) + 0.60(0.90)(−2) = +0.30

Precisely: this reweights actions at the states saved in the old batch. It does not recreate the different states a far-away new policy might visit. That remaining approximation is another reason PPO keeps the update proximal.

About the logarithm: −ln(0.40)=0.916 and −ln(0.46)=0.777, so the familiar chosen-action log gradient changes as probability changes. But the ratio is not an ad-hoc boost to repair the log curve. It is the bookkeeping identity that makes old samples represent the candidate policy’s new action mixture.
ratio = new_prob / old_prob

# Numerically stable code uses log probabilities:
ratio = torch.exp(new_log_prob - old_log_prob.detach())
Why ordinary −log(P) × advantage can still work

At the first update, new and old policies are identical, so the ratio is 1. The gradient of −ratio × advantage is then exactly the ordinary policy-gradient direction.

One fresh update per batch is vanilla policy gradient/A2C and is completely legitimate. PPO earns its extra machinery because it deliberately performs several updates on the same increasingly stale batch.

04

The compact formula, finally earned.

Exact PPO replaces the teaching log-loss contribution with the importance-weighted surrogate, then expresses the same one-sided brake using clamp and minimum.

Production-shaped PyTorch

ratio = torch.exp(
    new_log_probs - old_log_probs.detach()
)                                      # shape [B]

unclipped = ratio * advantages        # shape [B]
clipped = torch.clamp(
    ratio, 1 - epsilon, 1 + epsilon
) * advantages                       # shape [B]

actor_loss = -torch.min(unclipped, clipped).mean()
                                                   # scalar
Ratio
Reweight

Make old action samples represent the current candidate policy.

Clamp
Propose a plateau

Create a capped alternative at 0.8 or 1.2.

Minimum
Choose pessimistically

Stop only excessive improvement, never a wrong-direction correction.

Unfold the “fancy” clamp/min into the readable mask 
too_far_in_good_direction = (
    ((advantages > 0) & (ratio > 1.2)) |
    ((advantages < 0) & (ratio < 0.8))
)

active = ~too_far_in_good_direction

# Same actor gradient as PPO almost everywhere:
masked_surrogate_loss = -(
    ratio * advantages * active
).mean()

The standard clamp/min form keeps a continuous, meaningful surrogate value on the plateau. The explicit mask replaces that constant contribution with zero; its actor gradient is the same away from the boundary.

One flashcard, several epochs.

The saved advantage does not get recomputed inside the PPO update. The policy changes; the evidence stays frozen.

Illustrative positive-advantage trajectory

# On the next forward pass, this sample lands on the plateau.

Not a hard probability constraint

PPO does not project the probability back to 0.48. It removes this sample’s incentive after the boundary. A finite step can overshoot.

Not necessarily frozen afterward

Other samples, entropy loss, value loss, and shared parameters may still move this probability. Only this sample’s clipped actor contribution is flat.

Why several epochs still help

Different samples reach their plateaus at different times. An early-clipped sample contributes no further actor gradient while other samples continue learning. After the chosen number of epochs—or an early KL stop—the batch is discarded and fresh experience is collected.

06

Why not directly force 0.40 → 0.48?

Because a desired output change is not a loss derivative, and backpropagation is not an inverse solver.

Your clean counterexample

L = 2y     y = (x−1)²     x = 5

Currently, y=16, L=32, dL/dy=2, dy/dx=8, and therefore dL/dx=16.

Fantasy: optimize y directly

Choose learning rate 8:

y′ = 16 − 8(2) = 0
Reality: optimizer changes x

Reuse learning rate 8:

x′ = 5 − 8(16) = −123

Then y′=(−123−1)²=15376. The output explodes instead of reaching zero.

To make y=0, we need x=1. For this isolated quadratic and starting point, a specially solved parameter-space step would use learning rate 0.25: 5−0.25(16)=1. That is solving for this target—not ordinary backpropagation.

desired Δprobability ≠ dLoss / dProbability

The network shares parameters

One weight update changes probabilities for many states and all competing actions. A step that lands one sample exactly at 0.48 generally makes other samples miss their targets.

Even supervised learning moves gradually

A one-hot label does not directly set a class probability to 1.0. It defines a loss; gradient steps adjust shared weights over time. SFT does not bypass this geometry.

Could we design a target-based algorithm? Yes: construct a full target distribution, decide how to redistribute probability among other actions, and fit it with KL divergence or cross-entropy. That is a different constrained/projection-style algorithm, not PPO’s clipped surrogate.
07

The complete PPO loop.

Every line now has one job: collect under the old policy, freeze evidence, reuse it with importance ratios, and remove excessive incentives.

for iteration in range(num_iterations):
    # 1. Expensive: interact with the environment once.
    states, actions, returns, old_log_probs = collect(policy)

    # 2. Freeze what this old-policy batch taught us.
    with torch.no_grad():
        old_values = critic(states)
        advantages = returns - old_values
        advantages = (advantages - advantages.mean()) / (
            advantages.std() + 1e-8
        )

    # 3. Cheap: reuse the saved batch for a few epochs.
    for epoch in range(4):
        new_log_probs = policy_log_prob(states, actions)
        ratio = torch.exp(new_log_probs - old_log_probs.detach())

        unclipped = ratio * advantages
        clipped = torch.clamp(ratio, 0.8, 1.2) * advantages
        actor_loss = -torch.min(unclipped, clipped).mean()

        values = critic(states)
        critic_loss = mse_loss(values, returns)

        loss = actor_loss + 0.5 * critic_loss
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

Bridge to language models

For an LLM policy, the state is the prompt plus generated tokens, the action is the next token, and the policy probability is the probability assigned to the sampled token. The same old/new log-probability ratio and clipped advantage objective can be computed token by token. PPO’s conceptual job does not change.

Final mental model: Actor–Critic tells the policy which sampled choices were better or worse than expected. PPO lets the optimizer reread that evidence, but withdraws each sample’s permission to keep pushing once the policy has changed enough in the requested direction.

Primary reference: Schulman et al., “Proximal Policy Optimization Algorithms”.
