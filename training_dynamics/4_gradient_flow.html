Training Dynamics Ch4: Gradient Flow

Why deep networks are hard to train — and how residuals and normalization fix it

Vanishing Gradients — Layers That Stop Learning

Backprop multiplies gradients through layers

To update layer 1 in a 10-layer network, the gradient flows backward through all 10 layers. At each layer, the gradient is multiplied by the Jacobian of that layer. If each Jacobian has spectral norm less than 1, the gradient shrinks exponentially.

// Gradient at layer k = product of all Jacobians from output to layer k grad_at_layer_1 = J₁₀ × J₉ × J₈ × ... × J₁ × grad_output   // If each |Jₖ| ≈ 0.9 (slight shrinkage per layer): |grad at layer 1| ≈ 0.9^10 × |grad_output| = 0.349 × |grad_output| // 65% lost   // If each |Jₖ| ≈ 0.5 (more aggressive shrinkage): |grad at layer 1| ≈ 0.5^10 × |grad_output| = 0.001 × |grad_output| // 99.9% lost!

Gradient shrinkage by depth — norm=0.9 vs norm=0.5

Layer from outputGrad (norm=0.9)Grad (norm=0.5)Update size
Layer 10 (near output)1.0001.000Normal
Layer 80.9² = 0.8100.5² = 0.250Somewhat reduced
Layer 50.9⁵ = 0.5900.5⁵ = 0.031Much reduced
Layer 20.9⁸ = 0.4300.5⁸ = 0.004Tiny
Layer 1 (input)0.9¹⁰ = 0.3490.5¹⁰ = 0.001Effectively zero

With norm=0.5 at 10 layers: layer 1 gets a gradient 1000× smaller than layer 10. At lr=0.001, layer 10 moves by 0.001 while layer 1 moves by 0.000001. The early layers barely change — they're frozen in place from initialization.

Why sigmoid activations caused the original vanishing problem

// sigmoid(x) = 1 / (1 + exp(-x)) // gradient of sigmoid: σ'(x) = σ(x) × (1 − σ(x))   // At x=0 (best case): σ'(0) = 0.5 × 0.5 = 0.25 // max gradient is only 0.25!   // At x=3 (saturated): σ'(3) = 0.95 × 0.05 = 0.048 // nearly zero   // 10 sigmoid layers, all at x=0: grad multiplied = 0.25^10 = 0.000001 // vanished completely   // ReLU: gradient is 1 for positive inputs → no shrinkage from activation // This is why ReLU replaced sigmoid in deep networks (2012, AlexNet)

Exploding Gradients — Too Much Signal

The flip side: Jacobian norm > 1

// If each Jacobian has spectral norm 1.1: |grad at layer 1| = 1.1^10 × |grad_output| = 2.59 × |grad_output| // 2.6× amplified   // If each Jacobian has spectral norm 1.5: |grad at layer 1| = 1.5^10 × |grad_output| = 57.7 × |grad_output| // 58× amplified!   // 20-layer network with norm 1.5: |grad at layer 1| = 1.5^20 = 3325 × |grad_output| // catastrophic

What exploding gradients look like in practice

// Training step with gradient explosion: loss_before = 1.32 grad_norm = 847.5 // should be ~1.0 for healthy training weight update: w -= 0.001 × 847.5 = w − 0.8475 // massive jump loss_after = 9.74 // jumped from 1.32 to 9.74 in one step   // Loss might recover, or the model might diverge to NaN

Fix: Gradient clipping — the standard solution

// Global gradient norm clipping (PyTorch) loss.backward() torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0) optimizer.step()   // Concrete: grad_norm = 847.5, max_norm = 1.0 scale = min(1.0, max_norm / grad_norm) = min(1.0, 1.0 / 847.5) = 0.00118   // All gradients scaled by 0.00118: new total grad norm = 847.5 × 0.00118 = 1.0 // capped at max_norm weight update = 0.001 × 1.0 = 0.001 // safe

Monitoring grad norm as a diagnostic

// Log grad_norm every step. Residual Connections — The Architectural Fix

The residual block

// Standard layer: output = F(x) // Residual layer: output = x + F(x) ← add input back   // The key insight: the gradient through a residual block d(x + F(x))/dx = 1 + dF(x)/dx   // The +1 term means gradient is ALWAYS at least 1, regardless of F // Even if F learns nothing (dF/dx ≈ 0), gradient flows as: grad_in = grad_out × (1 + 0) = grad_out // perfect passthrough

50-layer network: with vs without residuals

// Without residuals: each layer has Jacobian norm 0.9 grad at layer 1 = 0.9^50 × grad_output = 0.005 × grad_output // 99.5% gone   // With residuals: each block's gradient passes through the +1 highway d(x + F(x))/dx ≥ 1 // at minimum grad at layer 1 = grad_output × Π(1 + dF_k/dx) ≥ 1.0 × grad_output // at minimum, never vanishes   // This is why ResNets can be 152 layers (2015) // This is why Transformers (with residuals) can be 100+ layers

What residual blocks look like in a transformer

Residuals also improve the forward pass

// Early in training, F(x) is random noise. // Without residual: output = F(x) = random garbage // With residual: output = x + F(x) = x + small_noise ≈ x   // A 12-layer transformer at initialization outputs ≈ the input // (modulated by embeddings and positional encodings) // This is a reasonable starting point — not random garbage // Training then gradually builds up the F(x) terms

Weight Initialization — Starting in the Right Place

Why initialization matters

Before seeing any data, a randomly initialized network should produce approximately unit-variance activations at every layer. Too large → activations explode. Too small → activations vanish. Both cause vanishing/exploding gradients from step 1.

// Linear layer: y = Wx + b (no bias for this analysis) // x has n_in inputs, each with variance σ²_x Var(y_j) = n_in × Var(W_ij) × Var(x_i) // each weight × input adds variance   // Want Var(y) = Var(x) = 1 (preserve variance) n_in × Var(W) = 1 → Var(W) = 1/n_in → std(W) = 1/sqrt(n_in)

Xavier / Glorot initialization

// For tanh and linear activations (symmetric) // Balance variance for both forward pass (fan_in) and backward pass (fan_out) std = sqrt(2 / (fan_in + fan_out))   // Example: Linear(512 → 512) std = sqrt(2 / (512 + 512)) = sqrt(2 / 1024) = sqrt(0.00195) = 0.0442   // Example: Linear(768 → 3072) — typical transformer FFN expand layer std = sqrt(2 / (768 + 3072)) = sqrt(2 / 3840) = sqrt(0.000521) = 0.0228   // Example: Linear(3072 → 768) — FFN contract layer std = sqrt(2 / (3072 + 768)) = sqrt(2 / 3840) = 0.0228 // same (symmetric)

Kaiming / He initialization — for ReLU networks

// ReLU zeros out half the activations → halves the variance // Need to double the initial variance to compensate std = sqrt(2 / fan_in) // "He init" — note: 2, not 1   // Example: Conv layer with 64 input channels, 3×3 kernel fan_in = 64 × 3 × 3 = 576 std = sqrt(2 / 576) = sqrt(0.00347) = 0.0589   // Without He init: Xavier std = sqrt(2 / (576+64)) = 0.0559 // Similar but Xavier doesn't account for ReLU's variance halving // After 10 ReLU layers: Xavier → var × 0.5^10 = var × 0.001 (vanished) // After 10 ReLU layers: He → var × 1.0^10 = var × 1.0 (preserved)

What bad initialization causes

// Too large: std=1.0 for Linear(512,512) // Expected output std after layer = sqrt(512) × 1.0 = 22.6 // After 2 layers: 22.6² ≈ 511 // After 4 layers: 511² ≈ 261,000 → NaN in forward pass   // Too small: std=0.001 for Linear(512,512) // Expected output std = sqrt(512) × 0.001 = 0.0226 // After 2 layers: 0.0226² ≈ 0.0005 // After 4 layers: vanished → all activations ≈ 0 → dead network   // With Xavier: std = 0.0442 for Linear(512,512) // Output std after each layer ≈ 1.0 (by design) // Stable for any depth

Gradient Norm as a Training Diagnostic

What healthy grad norms look like

Log the total gradient norm every step (before clipping). This single number tells you a lot about training health.

// Compute total grad norm: total_norm = sqrt(sum(p.grad.data.norm(2)**2 for p in model.parameters()))   // Healthy training: grad_norm between 0.5 and 2.0 most steps step 100: grad_norm = 1.24 ✓ step 500: grad_norm = 0.87 ✓ step 1000: grad_norm = 1.43 ✓ step 2000: grad_norm = 0.65 ✓ decreasing is normal as loss drops

Reading the symptoms

0.5 – 2.0 (stable)HealthyNothing

Per-layer grad norms — finding which layers are broken

// Per-layer analysis reveals where the problem is for name, param in model.named_parameters(): if param.grad is not None: layer_norm = param.grad.data.norm(2).item() print(f"{name}: {layer_norm:.4f}")   // Example output revealing vanishing gradients: transformer.layer11.attn.weight: 1.432 // near output: normal transformer.layer8.attn.weight: 0.821 // still ok transformer.layer4.attn.weight: 0.083 // getting small transformer.layer1.attn.weight: 0.002 ✗ vanished — this layer barely learns embedding.weight: 0.0001 ✗ embedding layer frozen by vanishing

The full picture — what protects gradient flow in a modern LLM

TechniquePreventsHow
Residual connectionsVanishing+1 gradient highway at every layer
LayerNormExploding + VanishingNormalizes activations; stabilizes Jacobian
Kaiming/Xavier initBothUnit variance forward and backward
Grad clipping (1.0)ExplodingCaps total gradient norm after explosion

All five work together. Remove any one and training a 12+ layer network becomes unreliable. This is why the transformer architecture (residuals + LayerNorm + good init + AdamW + warmup) has become the universal default.

The residual connection math — one line to remember

d(x + F(x))/dx = 1 + dF(x)/dx   // The "1" is the gradient highway. // It guarantees gradient ≥ 1 even if F learns nothing. // This single equation is why 100+ layer networks are trainable.
