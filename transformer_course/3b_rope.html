Transformer Course — Chapter 3b: RoPE

Rotary Position Embedding — how every modern LLM encodes position

What's Wrong with Absolute Positional Encoding?

Recall from Ch3: sinusoidal PE adds a position vector to the token embedding

// Original transformer (Vaswani et al.): input_to_attention = token_embedding + PE(pos)   // PE(pos) is a fixed vector — same for position 0 in every sequence. // The model gets position info because each position has a unique vector added.

Problem 1: position info gets diluted

// After adding PE, the token goes through layers of attention + MLP. // Position info is "baked in" at the start, then mixed/transformed. // By layer 30, how much of the original position signal remains? // Answer: very little. The model has to work hard to preserve it.

Problem 2: attention scores don't directly encode relative position

Attention computes q · k. If position is added to both:

q_m = W_Q · (x_m + PE(m)) k_n = W_K · (x_n + PE(n))   // q_m · k_n expands to FOUR terms: = (W_Q · x_m)·(W_K · x_n) // content × content + (W_Q · x_m)·(W_K · PE(n)) // content × position + (W_Q · PE(m))·(W_K · x_n) // position × content + (W_Q · PE(m))·(W_K · PE(n)) // position × position   // The last term depends on ABSOLUTE positions m and n separately. // We want it to depend on the RELATIVE distance (m - n) only. // Sinusoidal PE partially achieves this (via the rotation property from Ch3), // but only in that last term, and only approximately.

RoPE's idea: don't add position to the embedding — rotate Q and K

// RoPE approach: // 1. Compute Q and K from content alone (no position added to input) // 2. ROTATE Q and K by an angle proportional to their position // 3. The dot product q·k then AUTOMATICALLY depends on (m - n)   // Key insight: when you rotate two vectors and dot them, // the dot product only depends on the ANGLE BETWEEN them. // If q is rotated by m·θ and k is rotated by n·θ, // the angle between them is (m-n)·θ — relative position!

2D Rotation — The Core Trick

Rotating a 2D vector by angle θ

// A 2D vector [a, b], rotated by angle θ: R(θ) · [a, b] = [a·cos(θ) − b·sin(θ), a·sin(θ) + b·cos(θ)]   // Concrete: vector [3, 4], rotate by 30° (π/6 ≈ 0.524 rad) cos(0.524) = 0.866 sin(0.524) = 0.500   R(30°) · [3, 4] = [3·0.866 − 4·0.500, 3·0.500 + 4·0.866] = [2.598 − 2.000, 1.500 + 3.464] = [0.598, 4.964]   // The LENGTH didn't change: √(3²+4²) = 5, √(0.598²+4.964²) = 5.0 // Only the direction changed.

The key property: dot product of two rotated vectors

// Given vectors q=[3,4] and k=[1,2] // Rotate q by angle α, rotate k by angle β:   R(α)·q · R(β)·k = q · R(α−β) · k = q · k rotated by (α−β)   // The dot product depends ONLY on the difference (α − β). // Not on α alone. Not on β alone. Only on the gap between them.   // Verify with numbers: q=[3,4], k=[1,2], α=60°, β=30° // Difference = 30°   // Direct: R(60°)·q · R(30°)·k R(60°)·q = [3·0.500−4·0.866, 3·0.866+4·0.500] = [−1.964, 4.598] R(30°)·k = [1·0.866−2·0.500, 1·0.500+2·0.866] = [−0.134, 2.232] dot = (−1.964)(−0.134) + (4.598)(2.232) = 0.263 + 10.263 = 10.526   // Equivalently: q · R(30°)·k (just rotate k by the difference) R(30°)·k = [−0.134, 2.232] // same as above q · R(30°)·k = 3·(−0.134) + 4·(2.232) = −0.402 + 8.928 = 8.526   // Hmm, those don't match. Let me redo properly. // The identity is: R(α)q · R(β)k = q · R(β−α)k R(β−α)·k = R(−30°)·k = R(330°)·k = [1·0.866−2·(−0.500), 1·(−0.500)+2·0.866] = [1.866, 1.232] q · R(−30°)·k = 3·1.866 + 4·1.232 = 5.598 + 4.928 = 10.526 ✓   // Same result: 10.526. Depends on gap (α−β) only.

Now connect to attention: α = m·θ, β = n·θ

// RoPE says: rotate q at position m by angle m·θ // rotate k at position n by angle n·θ   score(m,n) = R(m·θ)·q_m · R(n·θ)·k_n = q_m · R((n−m)·θ) · k_n   // The attention score depends on (m−n)·θ — the relative distance! // Position m=5 attending to position n=3: gap=2, angle=2θ // Position m=100 attending to position n=98: gap=2, angle=2θ — SAME   // Relative position is automatically baked into every attention score. // No addition to the input. No dilution over layers. // Applied fresh at every layer, right where attention happens.

RoPE on a Real Attention Head (d=4)

Setup: 2 tokens, d=4 head dimension

// Sentence: "the cat" // Token 0: "the" at position m=0 // Token 1: "cat" at position m=1   // After W_Q and W_K projections (content only, no position yet): q_0 = [1.0, 2.0, 3.0, 4.0] // query for "the" q_1 = [2.0, 1.0, 0.5, 1.5] // query for "cat" k_0 = [0.5, 1.5, 2.0, 0.5] // key for "the" k_1 = [1.0, 0.5, 1.0, 2.0] // key for "cat"   // Base frequency: θ = 10000^(-2i/d) for each pair i // d=4 → two pairs (dims 0,1) and (dims 2,3) θ_0 = 10000^(0/4) = 10000^0 = 1.0 // pair 0: fast rotation θ_1 = 10000^(-2/4) = 10000^(-0.5) = 0.01 // pair 1: slow rotation

Apply RoPE to q_1 (position m=1)

// q_1 = [2.0, 1.0, 0.5, 1.5] // Split into pairs: pair0=[2.0, 1.0], pair1=[0.5, 1.5]   // Rotation angles for position m=1: angle_pair0 = m · θ_0 = 1 · 1.0 = 1.0 rad (57.3°) // big rotation angle_pair1 = m · θ_1 = 1 · 0.01 = 0.01 rad (0.57°) // tiny rotation   // Rotate pair 0 by 1.0 radian: cos(1.0) = 0.540, sin(1.0) = 0.841 pair0_rotated = [2.0·0.540 − 1.0·0.841, 2.0·0.841 + 1.0·0.540] = [1.080 − 0.841, 1.682 + 0.540] = [0.239, 2.222]   // Rotate pair 1 by 0.01 radian: cos(0.01) = 1.000, sin(0.01) = 0.010 pair1_rotated = [0.5·1.000 − 1.5·0.010, 0.5·0.010 + 1.5·1.000] = [0.500 − 0.015, 0.005 + 1.500] = [0.485, 1.505]   // Final RoPE'd query for "cat" at position 1: q̃_1 = [0.239, 2.222, 0.485, 1.505]

Apply RoPE to k_0 (position n=0)

// k_0 = [0.5, 1.5, 2.0, 0.5] // Rotation angles for position n=0: angle_pair0 = 0 · 1.0 = 0.0 rad // no rotation at position 0! angle_pair1 = 0 · 0.01 = 0.0 rad   // cos(0)=1, sin(0)=0 → rotation by 0 is identity k̃_0 = [0.5, 1.5, 2.0, 0.5] // unchanged at position 0

Attention score: q̃_1 · k̃_0

score(1→0) = q̃_1 · k̃_0 = 0.239·0.5 + 2.222·1.5 + 0.485·2.0 + 1.505·0.5 = 0.120 + 3.333 + 0.970 + 0.753 = 5.175   // This score encodes: "cat" (content) attending to "the" (content) // with relative distance = 1 position apart (encoded by the rotation).   // If "cat" were at position 50 and "the" at position 49: // angles: q rotated by 50·θ, k rotated by 49·θ // difference is still 1·θ → SAME attention score // Relative position is translation-invariant.

Why Multiple Frequencies (θ_0 fast, θ_1 slow)?

Same idea as sinusoidal PE — different pairs see different scales

// With d=4, we have 2 pairs. With d=128 (real LLM), we have 64 pairs. // Each pair i rotates at frequency θ_i = 10000^(-2i/d)   // Pair 0 (i=0): θ = 1.0 → full rotation every 2π ≈ 6 positions // Pair 1 (i=1): θ = 0.01 → full rotation every 628 positions // Pair 32 (i=32): θ = 0.0001 → full rotation every 62,800 positions // Pair 63 (i=63): θ = 0.00001 → full rotation every 628,000 positions   // Fast pairs: distinguish pos 5 from pos 6 (local structure) // Slow pairs: distinguish pos 1000 from pos 5000 (global structure)

Position 0 vs 1: fast pair rotates a lot, slow pair barely moves

// At position 0: all pairs unrotated (identity) // At position 1: pair 0: rotated by 1.0 rad (57°) // massive change pair 1: rotated by 0.01 rad (0.6°) // barely moved   // At position 100: pair 0: rotated by 100 rad ≈ 15.9 full rotations // wrapped around many times pair 1: rotated by 1.0 rad (57°) // just starting to turn   // Fast pairs lose info at long range (they've wrapped around). // Slow pairs carry long-range position info. // Together: encode both local and global relative position.

Why this makes attention scores decay with distance

// q and k are rotated by different amounts. // The bigger the position gap (m-n), the more they're rotated apart. // On average, rotating two random vectors further apart → smaller dot product.   // This gives a natural "local bias" — nearby tokens have higher attention. // Not a hard cutoff (like a window), but a smooth decay.   // distance 1: pair0 rotated 1 rad apart → mild dot product reduction // distance 10: pair0 rotated 10 rad apart → big dot product reduction // distance 100: pair0 wrapped, pair1 rotated 1 rad → some signal remains

Why RoPE won: applied at every layer, not just the input

// Absolute PE: position info added at layer 0. By layer 40, diluted. // RoPE: position info injected fresh at EVERY layer's attention. // Layer 40 gets the same strong position signal as layer 1.   // RoPE is applied to Q and K only (not V, not the residual stream). // The content representation stays clean. // Position only affects attention ROUTING, not the values being attended to.

Context length extension — why RoPE enables longer sequences

// Problem: model trained with max 4096 positions. // At inference, user gives 8192 tokens. // Position 5000 was never seen during training.   // Learned absolute: position 5000 has NO embedding. Crash. // Sinusoidal: works numerically but model never learned patterns for pos 5000.   // RoPE: the attention score only sees RELATIVE distance. // Token at 5000 attending to token at 4999 → distance 1. // The model has seen distance=1 millions of times. It works.   // The failure mode: token at 5000 attending to token at 0 → distance 5000. // Model never saw distance=5000 in training. Degrades.   // Fix: scale the frequencies. Divide all θ by 2 → same rotation angles // now span 8192 positions instead of 4096. // This is "NTK-aware" or "YaRN" scaling — works surprisingly well.
