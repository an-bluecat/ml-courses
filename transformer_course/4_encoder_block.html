1. LayerNorm — Why Not BatchNorm?

BatchNorm's problem for transformers

BatchNorm normalizes across the batch dimension for each feature. For sequences this creates two problems:

  • Variable length: different sentences have different lengths. Padding complicates statistics.
  • Small batch size: language models often use batch size 1-8. With tiny batches, mean/variance estimates are noisy.
  • Autoregressive generation: at inference, batch size is 1. BatchNorm statistics are meaningless.

LayerNorm: normalize each token independently

For each token, compute mean and std across its d_model features, then normalize:

μ = mean(x) // mean across the d_model features of this one token σ = std(x) // std across the d_model features of this one token x_norm = (x - μ) / (σ + ε) output = γ · x_norm + β // γ and β are learned scalars

No cross-batch or cross-position statistics. Works perfectly with any batch size, any sequence length.

Worked example: one token with d_model=4

x = [3, -1, 2, 0] // Step 1: compute mean μ = (3 + (-1) + 2 + 0) / 4 = 4/4 = 1.0 // Step 2: compute variance, then std var = [(3-1)² + (-1-1)² + (2-1)² + (0-1)²] / 4 = [4 + 4 + 1 + 1] / 4 = 10/4 = 2.5 σ = sqrt(2.5) = 1.581 // Step 3: normalize each element x_norm[0] = (3 - 1) / 1.581 = 2.0 / 1.581 = 1.265 x_norm[1] = (-1 - 1) / 1.581 = -2.0 / 1.581 = -1.265 x_norm[2] = (2 - 1) / 1.581 = 1.0 / 1.581 = 0.632 x_norm[3] = (0 - 1) / 1.581 = -1.0 / 1.581 = -0.632 x_norm = [1.265, -1.265, 0.632, -0.632] // mean ≈ 0, std ≈ 1 ✓ // Step 4: scale and shift (γ=1, β=0 initially) output = 1·x_norm + 0 = [1.265, -1.265, 0.632, -0.632]

2. Residual Connections

The formula

output = x + Sublayer(x) // The input x is added to the sublayer's output // Introduced by ResNet (2015), adopted by transformers

Why? Gradient flow

In a deep network without residuals, gradients must pass through every layer during backprop. Each layer multiplies the gradient by its Jacobian. After N layers:

// Gradient at layer 0 (without residuals): grad_0 = J_N · J_{N-1} · ... · J_1 · grad_N // If each Jacobian norm is 0.9: 0.9^96 ≈ 0.000005 → vanishes! // With residual: output = x + f(x), so gradient includes +1 path: d(output)/dx = 1 + d(f(x))/dx // The "1" is an identity shortcut — gradient can always flow // Even if d(f(x))/dx ≈ 0, gradient ≈ 1 still passes through

The "if MHA learned nothing" argument

Suppose MHA at some layer learned essentially nothing useful (W_Q, W_K, W_V ≈ 0). Then MHA(x) ≈ 0.

// Without residual: output = MHA(x) ≈ 0 // information destroyed! // With residual: output = x + MHA(x) ≈ x + 0 = x // information preserved!

Residuals make each layer's job easier: just learn the residual correction, not the full transformation. An "identity function" is trivially learnable (set all weights to 0).

Depth enabled by residuals

Without residuals: training reliably to 6 layers is hard, 12 layers extremely hard, 96 layers (GPT-3) essentially impossible with SGD.

With residuals: GPT-3 has 96 layers. PaLM has 118 layers. Residual connections are what make depth achievable.

3. Feed-Forward Network

Structure: two linear layers with a nonlinearity

FFN(x) = max(0, x · W_1 + b_1) · W_2 + b_2 // W_1: d_model × d_ff (e.g., 512 × 2048) // W_2: d_ff × d_model (e.g., 2048 × 512) // ReLU in original paper; GELU used in BERT and most modern models // Parameters: W_1 = 512×2048 = 1,048,576 W_2 = 2048×512 = 1,048,576 total = ~2M per block

Worked example: one token, tiny sizes

// d_model=2, d_ff=4 (illustrative) x = [0.5, -0.3] W_1 = [[1, 2, -1, 0.5], // 2×4 [0, 1, 2, -1 ]] // Step 1: x · W_1 h[0] = 0.5·1 + (-0.3)·0 = 0.50 h[1] = 0.5·2 + (-0.3)·1 = 0.70 h[2] = 0.5·(-1) + (-0.3)·2 = -1.10 h[3] = 0.5·0.5 + (-0.3)·(-1) = 0.55 h = [0.50, 0.70, -1.10, 0.55] // Step 2: ReLU — zero out negatives h_relu = [0.50, 0.70, 0.00, 0.55] // -1.10 zeroed // Step 3: project back W_2 = [[1, 0], [0, 1], [1, 0], [0, 1]] // 4×2 out[0] = 0.50·1 + 0.70·0 + 0.00·1 + 0.55·0 = 0.50 out[1] = 0.50·0 + 0.70·1 + 0.00·0 + 0.55·1 = 1.25 FFN(x) = [0.50, 1.25]

Why expand to 4× d_model?

MHA's job: mix information across positions. After MHA, token i has incorporated context from other tokens.

FFN's job: process each position independently. It applies the same transformation to every token's vector. This is where the model stores factual knowledge and does computation on individual token representations.

The 4× expansion (d_model → 4·d_model) creates a "thinking space" — more hidden dimensions = more capacity to represent complex token-level transformations. Modern LLMs often use 8/3 × d_model with SwiGLU activation instead of ReLU.

4. Pre-LN vs Post-LN

Post-LN (original paper)

x1 = LayerNorm(x + MHA(x)) x2 = LayerNorm(x1 + FFN(x1)) // LN applied AFTER the residual add

Problem: the residual path passes through LN. This disrupts the clean gradient highway. At the start of training, when layers aren't yet useful, gradients still go through LN's normalization. Unstable; often needs careful warmup scheduling (linear warmup for thousands of steps) to not diverge.

Pre-LN (GPT-2, modern standard)

x1 = x + MHA(LayerNorm(x)) x2 = x1 + FFN(LayerNorm(x1)) // LN applied BEFORE the sublayer

Benefit: the residual path is completely clean — the raw signal x passes through unchanged. Gradients always have a direct path to early layers. Training is more stable and requires less warmup. Nearly all post-2019 models (GPT-2, GPT-3, Llama, Mistral) use Pre-LN.

Gradient flow comparison

// Gradient of output w.r.t. x (simplified): // Post-LN: d(x2)/dx = d(LN(x1+FFN(x1)))/dx // Gradient must pass through LN — can be scaled/distorted // Pre-LN: d(x2)/dx = 1 + d(FFN(LN(x1)))/dx // The "1" is a clean gradient path — always at least 1 ✓

One more variant: RMSNorm

Modern models (Llama, Gemma) often replace LayerNorm with RMSNorm — normalizes by root-mean-square rather than mean-centered std. No mean subtraction needed.

RMSNorm(x) = x / sqrt(mean(x²) + ε) · γ // Faster (no mean computation), often same or better quality // Example: Llama-3, Mistral, Gemma all use RMSNorm
