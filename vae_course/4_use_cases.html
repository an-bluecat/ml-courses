VAE Course — Chapter 4: Use Cases

Why reconstruction produces better representations than supervised prediction

Why Autoencoder Bottleneck Is Better Than MLP Hidden Layer

The question you're asking

If you just want a compressed representation (embedding), why not take a hidden layer from an MLP trained on some label? Both have a bottleneck. What does reconstruction actually add?

MLP shortcut — supervised prediction can ignore most of the input

// MLP trained on "will user click this video?"
video_features → [512 hidden] → P(click)
 
// What the 512 hidden nodes need to encode:
// Only what predicts clicks — thumbnail appeal, title curiosity, maybe topic.
// Production quality? Irrelevant for CTR → discarded.
// Video length? Irrelevant for click → discarded.
// Specific topic details? Mostly irrelevant → compressed away.
 
// The hidden layer encodes "how clickable is this?" not "what IS this?"
// Useful for one task. Brittle for anything else.

Autoencoder — reconstruction can't throw anything away

// Autoencoder trained to reconstruct video features:
video_features → [bottleneck: 32] → reconstruct video_features
 
// What the bottleneck must encode:
// Everything. If it drops production quality, reconstruction fails.
// If it drops topic details, reconstruction fails.
// If it drops video length, reconstruction fails.
 
// The bottleneck encodes "what IS this video, fully"
// No single task can bias it. No signal can dominate.
// Result: a general-purpose representation of the content.

Concrete: same video, two embeddings

// Video: "Advanced calculus lecture, 2 hours, low-budget camera, niche topic"
 
// MLP (click-prediction) embedding:
// CTR is low (niche topic, bad thumbnail) → embedding ≈ "not clickable"
// Two completely different calculus lectures get similar embeddings
// because they're both "not clickable." Their content differences are lost.
 
// Autoencoder embedding:
// Must encode: topic=calculus, depth=advanced, length=2h, style=lecture
// Two calculus lectures get similar embeddings (same content)
// A cooking video gets a very different embedding
// The content is what determines similarity, not clickability

Reconstruction forces generality. Supervised prediction allows shortcuts.

Anomaly Detection — Catch the Unusual

The idea

Train an autoencoder on normal data only. It learns to reconstruct normal inputs well. When an unusual input arrives, reconstruction error is high — it can't rebuild it from the bottleneck.

// Credit card transactions (training on normal ones):
normal_tx: [amount=50, merchant=grocery, time=18:00, country=US]
// Autoencoder trained for 1M normal transactions:
reconstruction error = 0.003 // very low, seen this pattern many times
 
suspicious_tx: [amount=4999, merchant=crypto, time=03:17, country=RU]
reconstruction error = 0.847 // HIGH — bottleneck can't compress this pattern
// The bottleneck learned to represent "normal transaction space"
// This transaction is outside that space → poor reconstruction → flag it

Why not just train a classifier "normal vs fraud"?

// Classifier problem: you need labeled fraud examples to train on.
// Fraud patterns change constantly — today's fraud doesn't look like last year's.
// A classifier trained on old fraud misses new attack patterns.
 
// Autoencoder advantage: only needs NORMAL data (abundant).
// Any deviation from normal — whether it's a known fraud pattern or a new one
// — shows up as high reconstruction error.
// Catches novel anomalies the classifier was never trained on.

Denoising Autoencoders — The Ancestor of Diffusion

The twist: corrupt the input, reconstruct the clean version

// Standard autoencoder:
input [1,1,1,1] → encoder → z → decoder → reconstruct [1,1,1,1]
 
// Denoising autoencoder:
clean input [1,1,1,1]
↓ add noise
noisy input [1.2, 0.7, 1.1, 0.8] → encoder → z → decoder → reconstruct [1,1,1,1]
// ^^^^^^^^^^^
// CLEAN version, not noisy!
// Loss = MSE(output, CLEAN input) ← must learn to undo the noise

Why this produces better representations

// Standard AE: just compresses. Can memorize pixel patterns.
// Denoising AE: must learn WHAT the input IS, not just compress it.
// To remove noise from a cat photo, you need to understand "cat" — not memorize pixels.
// Forces learning of robust, semantic features.
 
// This is the foundational idea behind diffusion models (Chapter 1 of diffusion_course):
// Add noise in 1000 steps. Train a network to reverse each step (denoise).
// Denoising autoencoder = diffusion model with 1 noise step instead of 1000.

Semantic IDs — VQ-VAE for Recommendation

The problem: cold start

// Standard ID embedding in recommenders:
video_id = 48291 → embedding lookup → 64-dim vector
// Trained by: user engaged with video 48291 → update its embedding
 
// NEW video uploaded today: id = 99999
// Zero plays → embedding never updated → stays random initialization
// Recommender doesn't know what to do with it → video is buried
// This is the cold-start problem.

VQ-VAE solution: derive ID from content, not from engagement

// VQ-VAE = VAE with a quantized (discrete) bottleneck
// Instead of z being a continuous number, z is forced to be a discrete code
 
// Step 1: train VQ-VAE on item content features (text, image, audio)
item_features → encoder → continuous z = [0.3, 0.8, -0.1, 0.5]
↓ quantize: snap to nearest codebook entry
code = [2, 7, 3] ← SEMANTIC ID
↓
decoder(codes) → reconstruct item_features
// Same as VAE: reconstruction loss forces codes to capture full content
// Codes [2,7,3] are the discrete IDs that go into the recommender

The codebook — what the codes mean

// The codebook is a lookup table of learned "prototype" vectors
codebook[2] = [0.31, 0.77, -0.12] // a learned "direction" in feature space
codebook[7] = [0.82, -0.23, 0.44] // another direction
codebook[3] = [-0.11, 0.51, 0.79] // another direction
 
// Code 2 at level 1 might emerge as: "educational content"
// Code 7 at level 2 might emerge as: "advanced/technical depth"
// Code 3 at level 3 might emerge as: "long-form"
// → [2,7,3] = "educational, advanced, long-form"
// You didn't label these meanings. They emerged from reconstruction.

Why reconstruction is essential for cold start

// Old video: 10M plays, embedding well-trained by engagement signal
ID embedding: 64 numbers, refined by 10M gradient updates → very good
Semantic ID: [2, 7, 3] → lookup codebook entries → also good
 
// NEW video: 0 plays, uploaded 5 minutes ago
ID embedding: random → useless
Semantic ID: encoder(content_features) → [2, 7, 3]
// Same [2,7,3] as the old similar video!
// Recommender can treat the new video similarly to the old one.
// Zero engagement data needed. Content IS the ID.
 
// This works because the VQ-VAE was trained on reconstruction.
// It had to understand content to rebuild it.
// An MLP trained on clicks couldn't do this — it never saw zero-click videos.

Transfer Learning — Pre-train on Unlabeled, Fine-tune on Labels

The setup

// You have:
// 10,000,000 unlabeled images (cheap — just scrape the internet)
// 10,000 labeled images (expensive — humans had to annotate)
 
// Option A: train MLP only on 10k labeled images
// Option B: train autoencoder on 10M unlabeled → fine-tune encoder on 10k labels
 
// Option B wins. Why?
// The autoencoder encoder has seen 10M images.
// It already knows edges, textures, shapes, objects — all from reconstruction.
// Fine-tuning on 10k labels is easy — just teach it to use what it already knows.

Why this works: reconstruction learns general features

// Autoencoder trained on 10M cat+dog+car+sky+food photos:
// To reconstruct all of these, the encoder must learn:
// - edges and contours (needed for all images)
// - color regions (needed for all images)
// - textures: fur, metal, glass (needed for specific categories)
// - shapes: round, rectangular (needed to distinguish objects)
 
// These are EXACTLY the features a classifier needs.
// The autoencoder learned them for free, from unlabeled data.
// Fine-tuning = "take these features and map them to categories"

When to Use What

Use caseArchitectureWhy reconstruction helps
Anomaly detectionAutoencoderNormal data reconstructs well. Anomalies don't. High error = flag.
Denoising / robustnessDenoising AEForces model to understand content, not memorize noise.
Image / text generationVAESmooth latent space → sample anywhere → valid output.
Semantic IDs (rec sys)VQ-VAEReconstruction forces content understanding → works for cold-start items.
Transfer learningAE or VAEPre-train on unlabeled data → rich general features → fine-tune cheaply.
Structured latents (YouTube)Variational IBBottleneck + reconstruction + routing → interpretable, debiased latents.

The common thread: reconstruction forces the bottleneck to keep everything. A supervised MLP can shortcut — it only keeps what its label needs. An autoencoder can't shortcut — it needs to rebuild the full input, so the bottleneck ends up being a general-purpose summary.
